# Packet Tracer – Configure Router Interfaces

> **Objectif** : Configurer les adresses IPv4 sur R1 et ses LANs, puis les adresses IPv6 sur R2 et ses LANs, et vérifier la connectivité entre les machines.

---

## Topologie

| Device | Interface | Adresse IP / Préfixe      | Passerelle par défaut |
|--------|-----------|---------------------------|-----------------------|
| R1     | G0/0      | 172.16.20.1 /25           | N/A                   |
| R1     | G0/1      | 172.16.20.129 /25         | N/A                   |
| R1     | S0/0/0    | 209.165.200.225 /30       | N/A *(déjà configuré)*|
| PC1    | NIC       | 172.16.20.10 /25          | 172.16.20.1           |
| PC2    | NIC       | 172.16.20.138 /25         | 172.16.20.129         |
| R2     | G0/0      | 2001:db8:c0de:12::1/64    | N/A                   |
| R2     | G0/1      | 2001:db8:c0de:13::1/64    | N/A                   |
| R2     | S0/0/1    | 2001:db8:c0de:11::1/64    | N/A *(déjà configuré)*|
| R2     | S0/0/1    | fe80::2 *(link-local)*    | N/A *(déjà configuré)*|
| PC3    | NIC       | 2001:db8:c0de:12::a/64    | fe80::2               |
| PC4    | NIC       | 2001:db8:c0de:13::a/64    | fe80::2               |

> 🔑 Mot de passe User EXEC : `cisco` | Mot de passe Privileged EXEC : `class`

---

## Partie 1 : Configurer l'adressage IPv4 et vérifier la connectivité

### Étape 1a – Configurer les interfaces LAN de R1

On accède à R1 via la console et on configure les deux interfaces Gigabit.  
Le `/25` correspond au masque `255.255.255.128`.  
Sans `no shutdown`, l'interface reste administrativement désactivée et ne transmet rien.

```
R1> enable
```
Mot de passe : `class`

```
R1# configure terminal

R1(config)# interface g0/0
R1(config-if)# ip address 172.16.20.1 255.255.255.128
R1(config-if)# description LAN PC1
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# interface g0/1
R1(config-if)# ip address 172.16.20.129 255.255.255.128
R1(config-if)# description LAN PC2
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# end
```

Vérifier que les interfaces sont bien **up/up** :

```
R1# show ip interface brief
```

### Étape 1b – Configurer PC1 et PC2

Les PCs se configurent via l'interface graphique de Packet Tracer :  
**Cliquer sur le PC → Desktop → IP Configuration**

**PC1 :**
- IP Address : `172.16.20.10`
- Subnet Mask : `255.255.255.128`
- Default Gateway : `172.16.20.1`

**PC2 :**
- IP Address : `172.16.20.138`
- Subnet Mask : `255.255.255.128`
- Default Gateway : `172.16.20.129`

### Étape 1c – Vérifier la connectivité IPv4

Depuis **PC1**, ouvre le Command Prompt (Desktop → Command Prompt) et teste :

```
ping 172.16.20.138
ping 172.16.20.129
```

Depuis **PC2** :

```
ping 172.16.20.10
ping 172.16.20.1
```

Les 4 pings doivent répondre avec succès ✅

## Partie 2 : Configurer l'adressage IPv6 et vérifier la connectivité

### Étape 2a – Activer le routage IPv6 et configurer les interfaces de R2

En IPv6, il faut d'abord activer explicitement le routage unicast sur le routeur.  
Sans `ipv6 unicast-routing`, le routeur ne transfèrera pas les paquets IPv6 entre ses interfaces.

```
R2> enable
```
Mot de passe : `class`

```
R2# configure terminal

R2(config)# ipv6 unicast-routing

R2(config)# interface g0/0
R2(config-if)# ipv6 address 2001:db8:c0de:12::1/64
R2(config-if)# description LAN PC3
R2(config-if)# no shutdown
R2(config-if)# exit

R2(config)# interface g0/1
R2(config-if)# ipv6 address 2001:db8:c0de:13::1/64
R2(config-if)# description LAN PC4
R2(config-if)# no shutdown
R2(config-if)# exit

R2(config)# end
```

Vérifier les interfaces IPv6 :

```
R2# show ipv6 interface brief
```

### Étape 2b – Configurer PC3 et PC4

**Cliquer sur le PC → Desktop → IP Configuration → IPv6**

**PC3 :**
- IPv6 Address : `2001:db8:c0de:12::a`
- Prefix Length : `64`
- Default Gateway : `fe80::2`

**PC4 :**
- IPv6 Address : `2001:db8:c0de:13::a`
- Prefix Length : `64`
- Default Gateway : `fe80::2`

> 💡 La passerelle est une adresse **link-local** (`fe80::2`), ce qui est normal en IPv6 — les passerelles par défaut utilisent toujours l'adresse link-local de l'interface du routeur.

### Étape 2c – Vérifier la connectivité IPv6

Depuis **PC3** (Desktop → Command Prompt) :

```
ping 2001:db8:c0de:13::a
```

Depuis **PC4** :

```
ping 2001:db8:c0de:12::a
```

Les pings doivent répondre avec succès ✅

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `ip address <ip> <masque>` | Assigne une adresse IPv4 à une interface |
| `ipv6 address <ip>/<préfixe>` | Assigne une adresse IPv6 à une interface |
| `ipv6 unicast-routing` | Active le routage IPv6 sur le routeur |
| `no shutdown` | Active une interface (désactivée par défaut) |
| `description <texte>` | Ajoute une description à l'interface |
| `show ip interface brief` | Affiche l'état et les IP de toutes les interfaces |
| `show ipv6 interface brief` | Affiche l'état et les IPv6 de toutes les interfaces |

---

> ⚠️ **Rappel** : Les interfaces des routeurs Cisco sont **shutdown par défaut**. Ne pas oublier `no shutdown` à chaque configuration d'interface, sinon la connectivité ne fonctionnera pas.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── Routeur R1 (IPv4) ────────────────────────
! ══════════════════════════════════════════════

    ! ─── User EXEC (mot de passe : cisco) ─────
    enable

        ! ─── Privileged EXEC (mdp : class) ────
        configure terminal

            ! ─── Global Config ────────────────
            interface g0/0
                ! ─── Interface Config ─────────
                ip address 172.16.20.1 255.255.255.128
                description LAN PC1
                no shutdown
                exit

            interface g0/1
                ! ─── Interface Config ─────────
                ip address 172.16.20.129 255.255.255.128
                description LAN PC2
                no shutdown
                exit

            end

        show ip interface brief

! ─── PC1 : configuration via GUI ───────────────
! Desktop > IP Configuration
!   IP      : 172.16.20.10
!   Masque  : 255.255.255.128
!   Gateway : 172.16.20.1

! ─── PC2 : configuration via GUI ───────────────
! Desktop > IP Configuration
!   IP      : 172.16.20.138
!   Masque  : 255.255.255.128
!   Gateway : 172.16.20.129

! ─── Vérification depuis PC1 ───────────────────
ping 172.16.20.138
ping 172.16.20.129

! ─── Vérification depuis PC2 ───────────────────
ping 172.16.20.10
ping 172.16.20.1


! ══════════════════════════════════════════════
! ─── Routeur R2 (IPv6) ────────────────────────
! ══════════════════════════════════════════════

    ! ─── User EXEC (mot de passe : cisco) ─────
    enable

        ! ─── Privileged EXEC (mdp : class) ────
        configure terminal

            ! ─── Global Config ────────────────
            ipv6 unicast-routing

            interface g0/0
                ! ─── Interface Config ─────────
                ipv6 address 2001:db8:c0de:12::1/64
                description LAN PC3
                no shutdown
                exit

            interface g0/1
                ! ─── Interface Config ─────────
                ipv6 address 2001:db8:c0de:13::1/64
                description LAN PC4
                no shutdown
                exit

            end

        show ipv6 interface brief

! ─── PC3 : configuration via GUI ───────────────
! Desktop > IP Configuration > IPv6
!   IPv6    : 2001:db8:c0de:12::a
!   Préfixe : 64
!   Gateway : fe80::2

! ─── PC4 : configuration via GUI ───────────────
! Desktop > IP Configuration > IPv6
!   IPv6    : 2001:db8:c0de:13::a
!   Préfixe : 64
!   Gateway : fe80::2

! ─── Vérification depuis PC3 ───────────────────
ping 2001:db8:c0de:13::a

! ─── Vérification depuis PC4 ───────────────────
ping 2001:db8:c0de:12::a
```
