# Packet Tracer – Implement a Small Network

> **Objectif** : Construire une petite topologie réseau from scratch dans Packet Tracer, câbler les équipements, configurer le routeur, les deux switches et les PCs, puis vérifier la connectivité.

---

## Topologie complétée

| Device | Interface | Adresse IP    | Masque        | Passerelle  |
|--------|-----------|---------------|---------------|-------------|
| RTA    | G0/0      | 10.10.10.1    | 255.255.255.0 | N/A         |
| RTA    | G0/1      | 10.10.20.1    | 255.255.255.0 | N/A         |
| SW1    | VLAN1     | 10.10.10.2    | 255.255.255.0 | 10.10.10.1  |
| SW2    | VLAN1     | 10.10.20.2    | 255.255.255.0 | 10.10.20.1  |
| PC-1   | NIC       | 10.10.10.10   | 255.255.255.0 | 10.10.10.1  |
| PC-2   | NIC       | 10.10.20.10   | 255.255.255.0 | 10.10.20.1  |

> 💡 Les adresses de PC-1 et PC-2 peuvent être n'importe quelle IP disponible dans leur réseau respectif. Les valeurs ci-dessus sont un exemple valide.

---

## Partie 1 : Créer la topologie réseau

### Étape 1 – Ajouter les équipements

Dans la barre d'outils en bas de Packet Tracer :

1. Cliquer sur **Network Devices** → **Routers** → glisser un routeur **1941** dans la zone de travail
2. Cliquer sur **Switches** → glisser **deux switches 2960** dans la zone de travail
3. Cliquer sur **End Devices** → glisser **deux PCs** dans la zone de travail
4. Arranger les appareils de façon lisible

### Étape 2 – Nommer les appareils

> ⚠️ Les noms d'affichage doivent correspondre **exactement** au tableau d'adressage, sinon la notation automatique ne sera pas accordée.

- Cliquer sur le **label sous l'icône** (pas sur l'icône elle-même, sinon la fenêtre de config s'ouvre)
- Renommer chaque appareil : `RTA`, `SW1`, `SW2`, `PC-1`, `PC-2`

### Étape 3 – Câbler les appareils

Cliquer sur l'**éclair orange** dans la barre du bas → choisir le câble **Copper Straight-Through** (ligne noire diagonale).

| De      | Port  | Vers  | Port         |
|---------|-------|-------|--------------|
| RTA     | G0/0  | SW1   | G0/1         |
| RTA     | G0/1  | SW2   | G0/1         |
| SW1     | F0/1  | PC-1  | FastEthernet0|
| SW2     | F0/1  | PC-2  | FastEthernet0|

## Partie 2 : Configurer les appareils

### Étape 1 – Configurer le routeur RTA

#### a. Paramètres de base

On configure le hostname, le mot de passe enable secret (chiffré MD5), le mot de passe des lignes et le banner MOTD.  
`login` sur les lignes est nécessaire pour que le mot de passe soit demandé à la connexion.

```
RTA> enable
RTA# configure terminal

RTA(config)# hostname RTA
RTA(config)# enable secret Ciscoenpa55
RTA(config)# banner motd #Authorized Access Only!#

RTA(config)# line console 0
RTA(config-line)# password Ciscolinepa55
RTA(config-line)# login
RTA(config-line)# exit

RTA(config)# line vty 0 4
RTA(config-line)# password Ciscolinepa55
RTA(config-line)# login
RTA(config-line)# exit
```

#### b. Configurer les interfaces

On attribue les IPs, on ajoute des descriptions lisibles et on active chaque interface avec `no shutdown`.

```
RTA(config)# interface g0/0
RTA(config-if)# ip address 10.10.10.1 255.255.255.0
RTA(config-if)# description LAN SW1 - PC1
RTA(config-if)# no shutdown
RTA(config-if)# exit

RTA(config)# interface g0/1
RTA(config-if)# ip address 10.10.20.1 255.255.255.0
RTA(config-if)# description LAN SW2 - PC2
RTA(config-if)# no shutdown
RTA(config-if)# exit

RTA(config)# end
RTA# copy running-config startup-config
```

---

### Étape 2 – Configurer SW1

L'interface de gestion d'un switch est l'interface **VLAN1**.  
La commande `ip default-gateway` permet au switch d'envoyer du trafic vers des réseaux distants (notamment pour être administré depuis un autre réseau).

```
SW1> enable
SW1# configure terminal

SW1(config)# enable secret Ciscoenpa55

SW1(config)# line console 0
SW1(config-line)# password Ciscolinepa55
SW1(config-line)# login
SW1(config-line)# exit

SW1(config)# line vty 0 15
SW1(config-line)# password Ciscolinepa55
SW1(config-line)# login
SW1(config-line)# exit

SW1(config)# interface vlan 1
SW1(config-if)# ip address 10.10.10.2 255.255.255.0
SW1(config-if)# no shutdown
SW1(config-if)# exit

SW1(config)# ip default-gateway 10.10.10.1

SW1(config)# end
SW1# copy running-config startup-config
```

- IP Address : `10.10.10.10`

### Étape 5 – Vérifier la connectivité

Depuis **PC-1** (Desktop → Command Prompt), pinger tous les appareils :

```
ping 10.10.10.1

ping 10.10.20.1

```

```
ping 10.10.10.2
ping 10.10.10.10
```

Tous les pings doivent réussir ✅

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `hostname <nom>` | Définit le nom du routeur/switch |
| `enable secret <mdp>` | Mot de passe enable chiffré (MD5) |
| `banner motd #<texte>#` | Message d'accueil affiché à la connexion |
| `line console 0` / `line vty 0 15` | Accès aux lignes console et VTY |
| `password <mdp>` + `login` | Définit et active le mot de passe de ligne |
| `ip address <ip> <masque>` | Assigne une IP à une interface |
| `description <texte>` | Décrit l'interface |
| `no shutdown` | Active une interface |
| `interface vlan 1` | Interface de gestion d'un switch |
| `ip default-gateway <ip>` | Passerelle par défaut d'un switch |

---

> ⚠️ **Rappel** : Sur un switch, `ip default-gateway` est indispensable pour qu'il puisse communiquer avec des hôtes sur d'autres réseaux (ex: être pingé ou administré depuis un réseau distant).
