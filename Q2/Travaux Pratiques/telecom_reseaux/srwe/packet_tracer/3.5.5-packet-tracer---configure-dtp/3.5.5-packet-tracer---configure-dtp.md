# Packet Tracer – Configure DTP

> **Objectif** : Configurer des trunks statiques et dynamiques entre switches, comprendre les modes DTP, définir un VLAN natif non-default, et vérifier la connectivité end-to-end entre PCs du même VLAN.

---

## Topologie

| Device | Interface | Adresse IP    | Masque        |
|--------|-----------|---------------|---------------|
| PC1    | NIC       | 192.168.10.1  | 255.255.255.0 |
| PC2    | NIC       | 192.168.20.1  | 255.255.255.0 |
| PC3    | NIC       | 192.168.30.1  | 255.255.255.0 |
| PC4    | NIC       | 192.168.30.2  | 255.255.255.0 |
| PC5    | NIC       | 192.168.20.2  | 255.255.255.0 |
| PC6    | NIC       | 192.168.10.2  | 255.255.255.0 |

| S2     | VLAN 99   | 192.168.99.2  | 255.255.255.0 |
| S3     | VLAN 99   | 192.168.99.3  | 255.255.255.0 |

### VLANs

| VLAN | Nom        | Réseau              |
|------|------------|---------------------|
| 10   | Red        | 192.168.10.0/24     |
| 20   | Blue       | 192.168.20.0/24     |
| 30   | Yellow     | 192.168.30.0/24     |

### Assignation des ports (S2 et S3)

| Ports       | VLAN | PCs connectés       |
|-------------|------|---------------------|
| F0/1 – F0/8 | 10   | PC1 (S2), PC6 (S3)  |
| F0/9 – F0/16| 20   | PC2 (S2), PC5 (S3)  |
| F0/17– F0/24| 30   | PC3 (S2), PC4 (S3)  |

### Modes DTP des liens trunk

| Lien      | Mode S1              | Mode S2/S3         | Résultat  |
|-----------|----------------------|--------------------|-----------|
| S1 ↔ S2  | dynamic desirable    | dynamic auto       | Trunk ✅  |
| S1 ↔ S3  | trunk + nonegotiate  | trunk              | Trunk ✅  |

---

## Modes DTP — rappel

| Mode               | Comportement |
|--------------------|--------------|
| `dynamic auto`     | Attend passivement — trunk si l'autre côté initie |
| `dynamic desirable`| Initie activement la négociation trunk |
| `trunk`            | Force le trunk statiquement |
| `nonegotiate`      | Désactive DTP — aucun paquet DTP envoyé |

> 💡 **Règle** : `desirable` + `auto` = trunk. `auto` + `auto` = pas de trunk. `trunk` + `nonegotiate` = trunk statique sans négociation DTP.

## Partie 1 : Vérifier les VLANs existants

Les VLANs 99 (Management) et 999 (Native) sont déjà préconfigurés sur tous les switches.

```
S1> enable
S1# show vlan brief
```

Répéter sur **S2** et **S3**.

**Réponse — Quels VLANs sont configurés ?**  
VLAN 1 (default), VLAN 99 (Management), VLAN 999 (Native), et les VLANs système 1002–1005.

## Partie 2 : Créer les VLANs 10, 20, 30 sur S2 et S3

```
S2> enable
S2# configure terminal

S2(config)# vlan 10
S2(config-vlan)# name Red
S2(config-vlan)# exit

S2(config)# vlan 20
S2(config-vlan)# name Blue
S2(config-vlan)# exit

S2(config)# vlan 30
S2(config-vlan)# name Yellow
S2(config-vlan)# exit

S2(config)# end
S2# show vlan brief
```

Répéter les mêmes commandes sur **S3**.

## Partie 3 : Assigner les VLANs aux ports de S2 et S3

`interface range` permet de configurer un groupe de ports en une seule fois, ce qui est bien plus efficace que de les configurer un par un.

```
S2> enable
S2# configure terminal

S2(config)# interface range f0/1 - 8
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 10
S2(config-if-range)# exit

S2(config)# interface range f0/9 - 16
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 20
S2(config-if-range)# exit

S2(config)# interface range f0/17 - 24
S2(config-if-range)# switchport mode access
S2(config-if-range)# switchport access vlan 30
S2(config-if-range)# exit

S2(config)# end
```

Répéter les mêmes commandes sur **S3**.

**Test ping PC1 → PC6 :**

```
! Depuis PC1
ping 192.168.10.2
! → échoue ✗
```

**Réponse — Pourquoi le ping échoue ?**  
Les VLANs sont bien assignés aux ports, mais les liens Gigabit entre S1–S2 et S1–S3 ne sont pas encore configurés en trunk. Le trafic VLAN 10 ne peut pas traverser d'un switch à l'autre.

## Partie 4 : Configurer les trunks sur S1, S2 et S3

### a. Lien S1 ↔ S2 : trunk dynamique (dynamic desirable)

S2 est par défaut en `dynamic auto` — il attend passivement.  
En mettant S1 en `dynamic desirable`, S1 initie la négociation → le trunk s'établit automatiquement.

```
S1> enable
S1# configure terminal

S1(config)# interface g0/1
S1(config-if)# switchport mode dynamic desirable
S1(config-if)# exit
```

**Réponse — Résultat de la négociation S1 ↔ S2 ?**  
`desirable` (S1) + `auto` (S2) = **trunk** — S1 initie, S2 accepte. Le trunk est établi automatiquement.

Vérifier sur S2 :

```
S2# show interfaces trunk
! → G0/1 doit apparaître en mode trunking
```

**Réponse — Mode et statut du port G0/1 sur S2 ?**  
Mode : `auto` (DTP a négocié). Statut : `trunking`. Encapsulation : `n-802.1q` (négociée).

### b. Lien S1 ↔ S3 : trunk statique + désactivation DTP

`switchport mode trunk` force le trunk sans négociation.  
`switchport nonegotiate` désactive l'envoi de paquets DTP — recommandé en production pour éviter les attaques de VLAN hopping via DTP.

```
S1(config)# interface g0/2
S1(config-if)# switchport mode trunk
S1(config-if)# switchport nonegotiate
S1(config-if)# exit

S1(config)# end
```

Vérifier le statut DTP global :

```
S1# show dtp
! → "1 interfaces using DTP" (seulement G0/1, G0/2 a DTP désactivé)
```

Vérifier tous les trunks actifs :

```
S1# show interfaces trunk
! → G0/1 : mode desirable, G0/2 : mode on (statique)
! → Native VLAN = 1 pour l'instant → à corriger
```

**Réponse — Quel est le VLAN natif actuellement ?**  
VLAN 1 par défaut sur les deux trunks — il doit être changé en VLAN 999.

### c. Configurer VLAN 999 comme VLAN natif sur S1

```
S1# configure terminal

S1(config)# interface range g0/1 - 2
S1(config-if-range)# switchport trunk native vlan 999
S1(config-if-range)# exit

S1(config)# end
```

> ⚠️ Des messages `NATIVE_VLAN_MISMATCH` apparaissent jusqu'à ce que S2 et S3 soient aussi configurés avec VLAN 999 comme natif.

### d. Configurer VLAN 999 comme VLAN natif sur S2 et S3

**Sur S2 :**

```
S2> enable
S2# configure terminal

S2(config)# interface g0/1
S2(config-if)# switchport trunk native vlan 999
S2(config-if)# exit

S2(config)# end
```

**Sur S3 :**

```
S3> enable
S3# configure terminal

S3(config)# interface g0/2
S3(config-if)# switchport trunk native vlan 999
S3(config-if)# exit

S3(config)# end
```

### e. Vérifier et corriger la connectivité

```
S1# show interfaces trunk
S2# show interfaces trunk
S3# show interfaces trunk
S1# show vlan brief
S2# show vlan brief
S3# show vlan brief
```

**Réponse — Pourquoi le ping PC1 → PC6 échoue encore ?**  
S1 n'a pas les VLANs 10, 20, 30 dans sa base VLAN — ils n'ont été créés que sur S2 et S3. Le trunk transporte les VLANs, mais S1 doit aussi les connaître pour les laisser passer. Il faut créer les VLANs 10, 20 et 30 sur S1.

**Correction — Créer les VLANs manquants sur S1 :**

```
S1# configure terminal

S1(config)# vlan 10
S1(config-vlan)# name Red
S1(config-vlan)# exit

S1(config)# vlan 20
S1(config-vlan)# name Blue
S1(config-vlan)# exit

S1(config)# vlan 30
S1(config-vlan)# name Yellow
S1(config-vlan)# exit

S1(config)# end
```

## Partie 5 : Reconfigurer le trunk sur S3

### a. Vérifier l'état de G0/2 sur S3

```
S3# show interfaces trunk
! → vérifier mode et encapsulation de G0/2
```

**Réponse — Mode et encapsulation sur G0/2 de S3 ?**  
G0/2 est en mode `auto` (dynamic auto) par défaut — il n'a pas encore été configuré en trunk statique comme G0/2 sur S1 (`nonegotiate`). S1 a désactivé DTP sur G0/2, donc S3 ne reçoit plus de paquets DTP et ne peut pas négocier automatiquement.

### b. Configurer G0/2 sur S3 pour correspondre à S1

Puisque S1 G0/2 est en `trunk + nonegotiate`, S3 G0/2 doit aussi être en trunk statique.

```
S3# configure terminal

S3(config)# interface g0/2
S3(config-if)# switchport mode trunk
S3(config-if)# switchport nonegotiate
S3(config-if)# exit

S3(config)# end
```

**Réponse — Mode et encapsulation après le changement ?**  
Mode : `on` (trunk statique). Encapsulation : `802.1q`.

Vérifier la négociation :

```
S3# show interfaces g0/2 switchport
! → "Negotiation of Trunking: Off" (nonegotiate actif)
```

## Partie 6 : Vérifier la connectivité end-to-end

```
! Depuis PC1 (VLAN 10)
ping 192.168.10.2
! → réussit ✅ (PC6, même VLAN 10)

! Depuis PC2 (VLAN 20)
ping 192.168.20.2
! → réussit ✅ (PC5, même VLAN 20)

! Depuis PC3 (VLAN 30)
ping 192.168.30.2
! → réussit ✅ (PC4, même VLAN 30)
```

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show vlan brief` | Affiche VLANs et ports associés |
| `show interfaces trunk` | Affiche trunks actifs, modes, VLANs autorisés |
| `show interfaces <int> switchport` | Détails d'un port (mode, VLAN natif, négociation) |
| `show dtp` | Affiche le statut DTP global |
| `switchport mode dynamic desirable` | Initie activement la négociation trunk |
| `switchport mode dynamic auto` | Attend passivement (mode par défaut) |
| `switchport mode trunk` | Trunk statique forcé |
| `switchport nonegotiate` | Désactive l'envoi de paquets DTP |
| `switchport trunk native vlan <num>` | Définit le VLAN natif du trunk |
| `interface range f0/1 - 8` | Configure un groupe de ports simultanément |

---

> ⚠️ **À retenir** :
> - `auto` + `auto` = **pas de trunk** — l'un des deux côtés doit initier.
> - `desirable` + `auto` = **trunk** — le mode par défaut sur 2960 est `auto`.
> - `nonegotiate` désactive DTP — l'autre côté doit être en trunk **statique** sinon le lien ne monte pas.
> - Les VLANs doivent exister sur **tous les switches** traversés par le trunk pour être transmis correctement.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Vérification des VLANs ────────
! ══════════════════════════════════════════════

    ! ─── Sur S1, S2, S3 (répéter) ───────────
    enable
        show vlan brief
        ! → VLANs 1, 99, 999 déjà présents


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Création VLANs 10/20/30 ───────
! (Sur S2 et S3 — répéter sur chacun)
! ══════════════════════════════════════════════

    enable

        configure terminal

            ! ─── Global Config ───────────────
            vlan 10
                ! ─── VLAN Config ─────────────
                name Red
                exit

            vlan 20
                ! ─── VLAN Config ─────────────
                name Blue
                exit

            vlan 30
                ! ─── VLAN Config ─────────────
                name Yellow
                exit

            end

        show vlan brief


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Assignation des ports ─────────
! (Sur S2 et S3 — répéter sur chacun)
! ══════════════════════════════════════════════

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range f0/1 - 8
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                exit

            interface range f0/9 - 16
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 20
                exit

            interface range f0/17 - 24
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            end

! ─── Test (échoue encore, pas de trunk) ───────
! PC1 > ping 192.168.10.2  → échoue ✗


! ══════════════════════════════════════════════
! ─── PARTIE 4 : Configuration des trunks ──────
! ══════════════════════════════════════════════

! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport mode dynamic desirable
                ! → S2 (auto) accepte : trunk négocié
                exit

            interface g0/2
                ! ─── Interface Config ────────
                switchport mode trunk
                switchport nonegotiate
                ! → trunk statique, DTP désactivé
                exit

            interface range g0/1 - 2
                ! ─── Interface Config ────────
                switchport trunk native vlan 999
                exit

            end

        show dtp
        show interfaces trunk


! ─── Créer les VLANs manquants sur S1 ─────────

    configure terminal

        ! ─── Global Config ───────────────────
        vlan 10
            ! ─── VLAN Config ─────────────────
            name Red
            exit

        vlan 20
            ! ─── VLAN Config ─────────────────
            name Blue
            exit

        vlan 30
            ! ─── VLAN Config ─────────────────
            name Yellow
            exit

        end

    show vlan brief


! ─── Sur S2 ───────────────────────────────────

    enable

        show interfaces trunk
        ! → vérifier G0/1 en trunking (négocié par DTP)

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport trunk native vlan 999
                exit

            end


! ─── Sur S3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/2
                ! ─── Interface Config ────────
                switchport trunk native vlan 999
                exit

            end


! ══════════════════════════════════════════════
! ─── PARTIE 5 : Reconfigurer trunk S3 G0/2 ────
! ══════════════════════════════════════════════

    ! ─── Sur S3 ──────────────────────────────
    enable

        show interfaces trunk
        ! → G0/2 : mode auto → problème, S1 a nonegotiate

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/2
                ! ─── Interface Config ────────
                switchport mode trunk
                switchport nonegotiate
                exit

            end

        show interfaces g0/2 switchport
        ! → "Negotiation of Trunking: Off" ✅


! ══════════════════════════════════════════════
! ─── PARTIE 6 : Vérification connectivité ─────
! ══════════════════════════════════════════════

! PC1 > ping 192.168.10.2  → réussit ✅ (PC6)
! PC2 > ping 192.168.20.2  → réussit ✅ (PC5)
! PC3 > ping 192.168.30.2  → réussit ✅ (PC4)
```
