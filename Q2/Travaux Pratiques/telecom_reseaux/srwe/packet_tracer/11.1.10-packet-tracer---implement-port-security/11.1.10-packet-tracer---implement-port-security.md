# Packet Tracer – Implement Port Security

> **Objectif** : Configurer la sécurité de port sur un switch pour restreindre l'accès en fonction des adresses MAC — empêcher les appareils non autorisés de se connecter au réseau.

---

## Topologie

| Device       | Interface | Adresse IP  | Masque        |
|--------------|-----------|-------------|---------------|
| S1           | VLAN 1    | 10.10.10.2  | 255.255.255.0 |
| PC1          | NIC       | 10.10.10.10 | 255.255.255.0 |
| PC2          | NIC       | 10.10.10.11 | 255.255.255.0 |
| Rogue Laptop | NIC       | 10.10.10.12 | 255.255.255.0 |

### Modes de violation Port Security

| Mode       | Port désactivé ? | Log généré ? | Paquets inconnus |
|------------|-----------------|--------------|------------------|
| `shutdown` | ✅ Oui (err-disabled) | ✅ Oui | Droppés |
| `restrict` | ❌ Non          | ✅ Oui       | Droppés |
| `protect`  | ❌ Non          | ❌ Non       | Droppés |

> 💡 **Sticky MAC** : le switch apprend dynamiquement l'adresse MAC du premier appareil connecté et la sauvegarde dans la running-config — pas besoin de configurer manuellement chaque adresse.

---

## Partie 1 : Configurer la Port Security

### a–d. Activer la port security sur F0/1 et F0/2

```
S1> enable
S1# configure terminal

S1(config)# interface range f0/1 - 2
S1(config-if-range)# switchport port-security
! → active la port security sur les deux ports

S1(config-if-range)# switchport port-security maximum 1
! → un seul appareil autorisé par port

S1(config-if-range)# switchport port-security mac-address sticky
! → apprend et mémorise dynamiquement la MAC du premier appareil

S1(config-if-range)# switchport port-security violation restrict
! → en cas de violation : log + drop des paquets, port reste actif
S1(config-if-range)# exit
```

### e. Désactiver tous les ports inutilisés

```
S1(config)# interface range f0/3 - 24, g0/1 - 2
S1(config-if-range)# shutdown
S1(config-if-range)# exit

S1(config)# end
S1# copy running-config startup-config
```

## Partie 2 : Vérifier la Port Security

### a. Tester la connectivité initiale

```
! Depuis PC1
ping 10.10.10.11
! → réussit ✅ — les MACs de PC1 et PC2 sont apprises en sticky
```

### b. Vérifier les MACs apprises dans la running-config

```
S1# show run | begin interface
! → chercher les lignes :
!   switchport port-security mac-address sticky <MAC_PC1>  (sur F0/1)
!   switchport port-security mac-address sticky <MAC_PC2>  (sur F0/2)
```

### c. Afficher les informations de port security

```
S1# show port-security
! → affiche pour chaque port : max MACs, MACs actuelles, violations, action

S1# show port-security address
! → liste toutes les MACs sécurisées avec leur port et type (Sticky)
```

### d–e. Connecter le Rogue Laptop sur un port inutilisé

Connecter le Rogue Laptop sur un port F0/3–F0/24 → voyant **rouge** (port en shutdown).

Pour tester temporairement, activer le port :

```
S1# configure terminal

S1(config)# interface f0/3
S1(config-if)# no shutdown
S1(config-if)# exit

S1(config)# end
```

```
! Depuis Rogue Laptop
ping 10.10.10.10   ! → réussit ✅ (pas de port security sur F0/3)
ping 10.10.10.11   ! → réussit ✅
```

Après vérification, désactiver le port :

```
S1# configure terminal

S1(config)# interface f0/3
S1(config-if)# shutdown
S1(config-if)# exit

S1(config)# end
```

### f. Brancher le Rogue Laptop sur F0/2 (port sécurisé de PC2)

1. Débrancher PC2 de F0/2.
2. Brancher le Rogue Laptop sur F0/2.

```
! Depuis Rogue Laptop
ping 10.10.10.10
! → échoue ✗ — MAC du Rogue Laptop ≠ MAC sticky de PC2
! → violation restrict : paquets droppés, port reste actif
```

### g. Afficher les violations sur F0/2

```
S1# show port-security interface f0/2
! → Port Status      : Secure-up
! → Violation Mode   : Restrict
! → Security Violation Count : <nombre> violations
```

**Réponse — combien de violations ?**  
Autant de paquets qu'a envoyé le Rogue Laptop depuis sa connexion — chaque tentative de ping = plusieurs violations (ARP + ICMP).

### h. Rebrancher PC2 sur F0/2

1. Débrancher le Rogue Laptop.
2. Rebrancher PC2 sur F0/2.

```
! Depuis PC2
ping 10.10.10.10
! → réussit ✅
```

**Réponse — pourquoi PC2 peut pinguer PC1 mais pas le Rogue Laptop ?**  
Le switch a mémorisé la **MAC address de PC2** en sticky sur F0/2 après le premier ping réussi. Quand le Rogue Laptop (MAC différente) s'est connecté sur ce même port, sa MAC ne correspondait pas → violation `restrict` → paquets droppés. Quand PC2 est reconnecté, sa MAC correspond à l'entrée sticky → trafic autorisé normalement.

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `switchport port-security` | Active la port security sur un port d'accès |
| `switchport port-security maximum <n>` | Nombre max de MACs autorisées |
| `switchport port-security mac-address sticky` | Apprend et mémorise la MAC dynamiquement |
| `switchport port-security mac-address <MAC>` | Définit une MAC statique autorisée |
| `switchport port-security violation <mode>` | Définit l'action en cas de violation |
| `show port-security` | Vue globale de la port security sur tous les ports |
| `show port-security interface <int>` | Détail d'un port : statut, violations, MAC |
| `show port-security address` | Liste toutes les MACs sécurisées |
| `show run \| begin interface` | Vérifie les MACs sticky dans la config |

---

> ⚠️ **À retenir** :
> - La port security ne fonctionne que sur les **ports d'accès** (`switchport mode access`) — pas sur les trunks.
> - En mode `restrict`, le port reste actif mais les paquets non autorisés sont droppés et un compteur de violations s'incrémente.
> - En mode `shutdown` (défaut), le port passe en `err-disabled` — il faut faire `shutdown` puis `no shutdown` pour le réactiver.
> - Les MACs sticky sont sauvegardées dans la running-config mais **pas automatiquement** dans la startup-config — faire `copy run start` pour les persister après redémarrage.

---

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Configuration Port Security ───
! ══════════════════════════════════════════════

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range f0/1 - 2
                ! ─── Interface Config ────────
                switchport port-security
                switchport port-security maximum 1
                switchport port-security mac-address sticky
                switchport port-security violation restrict
                exit

            interface range f0/3 - 24, g0/1 - 2
                ! ─── Interface Config ────────
                shutdown
                exit

            end

        copy running-config startup-config


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Vérification ──────────────────
! ══════════════════════════════════════════════

! PC1 > ping 10.10.10.11  → ✅ (MACs apprises en sticky)

        show run | begin interface
        ! → voir MACs sticky sur F0/1 et F0/2

        show port-security
        show port-security address


! ─── Test Rogue Laptop sur port inutilisé ─────

        configure terminal

            ! ─── Global Config ───────────────
            interface f0/3
                ! ─── Interface Config ────────
                no shutdown
                exit

            end

! Rogue Laptop > ping 10.10.10.10  → ✅ (F0/3 sans port security)

        configure terminal

            ! ─── Global Config ───────────────
            interface f0/3
                ! ─── Interface Config ────────
                shutdown
                exit

            end


! ─── Test Rogue Laptop sur F0/2 (port sécurisé) ─
! Débrancher PC2, brancher Rogue Laptop sur F0/2
! Rogue Laptop > ping 10.10.10.10  → ✗ (violation restrict)

        show port-security interface f0/2
        ! → voir le compteur de violations

! Rebrancher PC2 sur F0/2
! PC2 > ping 10.10.10.10  → ✅ (MAC sticky de PC2 correspond)
```
