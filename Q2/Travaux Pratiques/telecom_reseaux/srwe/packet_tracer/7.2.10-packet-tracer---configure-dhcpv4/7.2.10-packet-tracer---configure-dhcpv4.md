# Packet Tracer – Configure DHCPv4

> **Objectif** : Configurer un routeur Cisco comme serveur DHCP, mettre en place un DHCP relay pour les réseaux distants, et configurer un routeur comme client DHCP vers l'ISP.

---

## Topologie

| Device     | Interface | Adresse IP      | Masque          | Passerelle    |
|------------|-----------|-----------------|-----------------|---------------|
| R1         | G0/0      | 192.168.10.1    | 255.255.255.0   | N/A           |
| R1         | S0/0/0    | 10.1.1.1        | 255.255.255.252 | N/A           |
| R2         | G0/0      | 192.168.20.1    | 255.255.255.0   | N/A           |
| R2         | G0/1      | DHCP (ISP)      | DHCP (ISP)      | N/A           |
| R2         | S0/0/0    | 10.1.1.2        | 255.255.255.252 | N/A           |
| R2         | S0/0/1    | 10.2.2.2        | 255.255.255.252 | N/A           |
| R3         | G0/0      | 192.168.30.1    | 255.255.255.0   | N/A           |
| R3         | S0/0/1    | 10.2.2.1        | 255.255.255.252 | N/A           |
| PC1        | NIC       | DHCP            | DHCP            | DHCP          |
| PC2        | NIC       | DHCP            | DHCP            | DHCP          |
| DNS Server | NIC       | 192.168.20.254  | 255.255.255.0   | 192.168.20.1  |

### Architecture DHCP

```
PC1 ── R1 (LAN 192.168.10.0/24)
        └── ip helper-address 10.1.1.2 ──► R2 (serveur DHCP)
                                               ├── pool R1-LAN (192.168.10.0/24)
                                               └── pool R3-LAN (192.168.30.0/24)
PC2 ── R3 (LAN 192.168.30.0/24)
        └── ip helper-address 10.2.2.2 ──► R2 (serveur DHCP)

R2 G0/1 ──► ISP (reçoit son IP en DHCP client)
```

---

## Partie 1 : Configurer R2 comme serveur DHCP

### Étape 1 – Exclure les adresses statiques des pools

Les adresses déjà assignées statiquement doivent être exclues du pool DHCP pour éviter les conflits.  
On exclut les 10 premières adresses de chaque réseau LAN (gateway + réserve pour serveurs/équipements).

```
R2> enable
R2# configure terminal

R2(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.10
! → exclut .1 (gateway R1) + .2 à .10 (réservés)

R2(config)# ip dhcp excluded-address 192.168.30.1 192.168.30.10
! → exclut .1 (gateway R3) + .2 à .10 (réservés)
```

### Étape 2 – Créer le pool DHCP pour le LAN de R1

`ip dhcp pool` crée un pool et entre en mode `dhcp-config`.  
`network` définit le réseau distribué. `default-router` = passerelle envoyée aux clients. `dns-server` = serveur DNS communiqué aux clients.

> ⚠️ Le nom du pool est **sensible à la casse** : `R1-LAN` ≠ `r1-lan`.

```
R2(config)# ip dhcp pool R1-LAN
R2(dhcp-config)# network 192.168.10.0 255.255.255.0
R2(dhcp-config)# default-router 192.168.10.1
R2(dhcp-config)# dns-server 192.168.20.254
R2(dhcp-config)# exit
```

### Étape 3 – Créer le pool DHCP pour le LAN de R3

```
R2(config)# ip dhcp pool R3-LAN
R2(dhcp-config)# network 192.168.30.0 255.255.255.0
R2(dhcp-config)# default-router 192.168.30.1
R2(dhcp-config)# dns-server 192.168.20.254
R2(dhcp-config)# exit

R2(config)# end
```

## Partie 2 : Configurer le DHCP Relay

### Pourquoi un relay ?

Le serveur DHCP est sur R2. Les clients (PC1, PC2) sont sur des LANs distants connectés à R1 et R3.  
Les messages DHCP Discover sont des **broadcasts** — ils ne traversent pas les routeurs par défaut.  
`ip helper-address` transforme ces broadcasts en **unicasts** vers l'adresse du serveur DHCP, permettant au trafic DHCP de traverser les routeurs.

> 💡 L'adresse helper doit pointer vers l'interface de **R2 la plus proche** de chaque routeur — soit l'interface série qui les relie.

### Étape 1 – Configurer le relay sur R1

```
R1> enable
R1# configure terminal

R1(config)# interface g0/0
R1(config-if)# ip helper-address 10.1.1.2
! → redirige les DHCP broadcasts du LAN vers R2 (S0/0/0 = 10.1.1.2)
R1(config-if)# exit

R1(config)# end
```

### Étape 2 – Configurer le relay sur R3

```
R3> enable
R3# configure terminal

R3(config)# interface g0/0
R3(config-if)# ip helper-address 10.2.2.2
! → redirige les DHCP broadcasts du LAN vers R2 (S0/0/1 = 10.2.2.2)
R3(config-if)# exit

R3(config)# end
```

### Étape 3 – Configurer PC1 et PC2 en DHCP

Via l'interface graphique : **cliquer sur le PC → Desktop → IP Configuration → DHCP**

PC1 et PC2 doivent recevoir automatiquement une IP depuis le pool correspondant :
- PC1 → pool `R1-LAN` → adresse dans `192.168.10.11` – `.254`
- PC2 → pool `R3-LAN` → adresse dans `192.168.30.11` – `.254`

---

## Partie 3 : Configurer R2 comme client DHCP

Un routeur peut aussi recevoir une IP en DHCP — utile pour les connexions vers l'ISP où l'adresse est assignée dynamiquement.  
`ip address dhcp` remplace la commande `ip address <ip> <masque>` habituelle.

```
R2> enable
R2# configure terminal

R2(config)# interface g0/1
R2(config-if)# ip address dhcp
R2(config-if)# no shutdown
R2(config-if)# exit

R2(config)# end
```

Utiliser **Fast Forward Time** pour accélérer l'attribution DHCP, puis vérifier :

```
R2# show ip interface brief
! → G0/1 doit afficher une IP assignée par l'ISP et statut UP/UP
```

---

## Partie 4 : Vérifier DHCP et connectivité

### Vérifier les baux DHCP actifs

```
R2# show ip dhcp binding
! → liste les IPs assignées avec l'adresse MAC du client
! Exemple attendu :
! 192.168.10.11  0002.4AA5.1470  --  Automatic  (PC1)
! 192.168.30.11  0004.9A97.2535  --  Automatic  (PC2)
```

### Vérifier la connectivité

```
! Depuis PC1
ping 192.168.30.11
! → réussit ✅ (PC2, réseau distant via DHCP relay)

ping 192.168.20.254
! → réussit ✅ (DNS Server)
```

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `ip dhcp excluded-address <début> <fin>` | Exclut une plage d'IPs du pool DHCP |
| `ip dhcp pool <nom>` | Crée un pool DHCP et entre en mode dhcp-config |
| `network <réseau> <masque>` | Définit le réseau distribué par ce pool |
| `default-router <ip>` | Passerelle envoyée aux clients DHCP |
| `dns-server <ip>` | Serveur DNS communiqué aux clients DHCP |
| `ip helper-address <ip>` | Relay DHCP — redirige les broadcasts vers le serveur |
| `ip address dhcp` | Configure l'interface comme client DHCP |
| `show ip dhcp binding` | Affiche les baux DHCP actifs |
| `show ip dhcp pool` | Affiche les informations des pools DHCP |
| `show ip interface brief` | Vérifie les IPs assignées sur les interfaces |

---

> ⚠️ **À retenir** :
> - Les noms des pools DHCP sont **sensibles à la casse**.
> - `ip helper-address` doit être configuré sur l'**interface LAN** du routeur relay (celle qui reçoit les broadcasts des clients), pas sur l'interface WAN.
> - L'adresse helper pointe vers **l'interface de R2 la plus proche** — l'IP de l'interface série entre les deux routeurs.
> - Exclure les adresses **avant** de créer les pools pour éviter les conflits.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : R2 comme serveur DHCP ─────────
! ══════════════════════════════════════════════

    ! ─── User EXEC ───────────────────────────
    enable

        ! ─── Privileged EXEC ─────────────────
        configure terminal

            ! ─── Global Config ───────────────
            ip dhcp excluded-address 192.168.10.1 192.168.10.10
            ip dhcp excluded-address 192.168.30.1 192.168.30.10

            ip dhcp pool R1-LAN
                ! ─── DHCP Config ─────────────
                network 192.168.10.0 255.255.255.0
                default-router 192.168.10.1
                dns-server 192.168.20.254
                exit

            ip dhcp pool R3-LAN
                ! ─── DHCP Config ─────────────
                network 192.168.30.0 255.255.255.0
                default-router 192.168.30.1
                dns-server 192.168.20.254
                exit

            end


! ══════════════════════════════════════════════
! ─── PARTIE 2 : DHCP Relay ────────────────────
! ══════════════════════════════════════════════

! ─── Sur R1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/0
                ! ─── Interface Config ────────
                ip helper-address 10.1.1.2
                exit

            end


! ─── Sur R3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/0
                ! ─── Interface Config ────────
                ip helper-address 10.2.2.2
                exit

            end


! ─── PCs : configuration via GUI ──────────────
! PC1 : Desktop > IP Configuration > DHCP
! PC2 : Desktop > IP Configuration > DHCP


! ══════════════════════════════════════════════
! ─── PARTIE 3 : R2 comme client DHCP ──────────
! ══════════════════════════════════════════════

    ! ─── Sur R2 ──────────────────────────────
    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                ip address dhcp
                no shutdown
                exit

            end

        show ip interface brief
        ! → G0/1 : IP assignée par ISP, UP/UP ✅


! ══════════════════════════════════════════════
! ─── PARTIE 4 : Vérification ──────────────────
! ══════════════════════════════════════════════

    ! ─── Sur R2 ──────────────────────────────
    enable

        show ip dhcp binding
        ! → PC1 : 192.168.10.11 (pool R1-LAN)
        ! → PC2 : 192.168.30.11 (pool R3-LAN)

! ─── Depuis PC1 ───────────────────────────────
ping 192.168.30.11   ! → ✅ (PC2)
ping 192.168.20.254  ! → ✅ (DNS Server)
```
