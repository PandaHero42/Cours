# Packet Tracer – Configure Router-on-a-Stick Inter-VLAN Routing

> **Objectif** : Configurer le routage inter-VLAN avec la méthode "Router-on-a-Stick" — un seul lien physique entre le switch et le routeur, divisé en sous-interfaces logiques pour chaque VLAN.

---

## Topologie

| Device | Interface | Adresse IP   | Masque        | Passerelle   |
|--------|-----------|--------------|---------------|--------------|
| R1     | G0/0.10   | 172.17.10.1  | 255.255.255.0 | N/A          |
| R1     | G0/0.30   | 172.17.30.1  | 255.255.255.0 | N/A          |
| PC1    | NIC       | 172.17.10.10 | 255.255.255.0 | 172.17.10.1  |
| PC3    | NIC       | 172.17.30.10 | 255.255.255.0 | 172.17.30.1  |

### Concept : Router-on-a-Stick

```
PC1 (VLAN 10) ──── S1 F0/11
PC3 (VLAN 30) ──── S1 F0/6
                   S1 G0/1 ───[trunk]─── R1 G0/0
                                              ├── G0/0.10 (VLAN 10) → 172.17.10.1
                                              └── G0/0.30 (VLAN 30) → 172.17.30.1
```

> 💡 **Router-on-a-Stick** = une seule interface physique sur le routeur, découpée en **sous-interfaces** (subinterfaces) virtuelles — une par VLAN. Le lien switch→routeur est un **trunk** qui transporte tous les VLANs. Chaque sous-interface agit comme passerelle pour son VLAN.

---

## Partie 1 : Ajouter les VLANs au switch S1

### Étape 1 – Créer les VLANs sur S1

```
S1> enable
S1# configure terminal

S1(config)# vlan 10
S1(config-vlan)# exit

S1(config)# vlan 30
S1(config-vlan)# exit

S1(config)# end
```

### Étape 2 – Assigner les ports aux VLANs

PC1 est connecté à **F0/11** (VLAN 10) et PC3 à **F0/6** (VLAN 30).

```
S1# configure terminal

S1(config)# interface f0/11
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 10
S1(config-if)# exit

S1(config)# interface f0/6
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 30
S1(config-if)# exit

S1(config)# end
S1# show vlan brief
```

### Étape 3 – Tester la connectivité initiale

```
! Depuis PC1
ping 172.17.30.10
! → échoue ✗
```

**Réponse — Pourquoi le ping échoue ?**  
PC1 (VLAN 10) et PC3 (VLAN 30) sont dans des VLANs et des sous-réseaux différents. Sans routeur inter-VLAN configuré, les deux machines ne peuvent pas communiquer — le switch seul ne peut pas router entre VLANs.

---

## Partie 2 : Configurer les sous-interfaces sur R1

### Étape 1 – Créer les sous-interfaces avec encapsulation 802.1Q

Une **sous-interface** est une interface logique créée sur une interface physique.  
La commande `encapsulation dot1Q <vlan>` associe la sous-interface à un VLAN spécifique — le routeur saura ainsi que les trames taggées VLAN 10 arrivant sur G0/0 doivent être traitées par G0/0.10.  
La notation `G0/0.10` est une convention — le numéro après le point correspond généralement au VLAN pour la lisibilité, mais ce n'est pas obligatoire.

```
R1> enable
R1# configure terminal

R1(config)# interface g0/0.10
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 172.17.10.1 255.255.255.0
R1(config-subif)# exit

R1(config)# interface g0/0.30
R1(config-subif)# encapsulation dot1Q 30
R1(config-subif)# ip address 172.17.30.1 255.255.255.0
R1(config-subif)# exit

R1(config)# end
```

### Étape 2 – Activer l'interface physique G0/0

Les sous-interfaces sont des interfaces virtuelles — elles dépendent de l'interface physique parente.  
Sans activer G0/0, toutes les sous-interfaces restent en état **down** même si elles sont bien configurées.

```
R1# show ip interface brief
! → G0/0.10 et G0/0.30 sont DOWN → G0/0 physique n'est pas activée

R1# configure terminal

R1(config)# interface g0/0
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# end

R1# show ip interface brief
! → G0/0, G0/0.10 et G0/0.30 sont maintenant UP/UP ✅
```

## Partie 3 : Tester et activer le routage inter-VLAN

### Étape 1 – Test ping (toujours en échec)

```
! Depuis PC1
ping 172.17.30.10
! → échoue encore ✗
```

**Réponse — Pourquoi le ping échoue encore ?**  
Le routeur est configuré, mais le lien entre S1 et R1 n'est pas encore en mode **trunk**. Le port G0/1 de S1 est encore en VLAN 1 par défaut — les trames taggées VLAN 10 et 30 ne peuvent pas traverser vers le routeur.

### Étape 2 – Activer le trunk sur S1 G0/1

Le port G0/1 connecte S1 au routeur R1. Il doit être en trunk pour transporter les trames taggées des deux VLANs vers les sous-interfaces correspondantes du routeur.

Vérifier d'abord quel VLAN G0/1 utilise :

```
S1# show vlan
! → G0/1 apparaît dans VLAN 1 (accès par défaut)
```

**Réponse — À quel VLAN G0/1 est-il assigné ?**  
VLAN 1 — le VLAN par défaut. Tant qu'il n'est pas en trunk, il ne transporte qu'un seul VLAN.

Configurer G0/1 en trunk :

```
S1# configure terminal

S1(config)# interface g0/1
S1(config-if)# switchport mode trunk
S1(config-if)# exit

S1(config)# end
```

**Réponse — Comment savoir qu'un port est trunk via `show vlan` ?**  
Un port trunk n'apparaît plus dans la liste des ports d'aucun VLAN dans `show vlan brief` — il n'est assigné à aucun VLAN d'accès. Sa disparition de la colonne "Ports" indique qu'il est en mode trunk.

Vérifier :

```
S1# show interfaces trunk
! → G0/1 doit apparaître en mode trunking
```

### Étape 3 – Vérifier la connectivité complète

```
! Depuis PC1
ping 172.17.10.1
! → réussit ✅ (passerelle VLAN 10)

ping 172.17.30.10
! → réussit ✅ (PC3, via routage inter-VLAN)

! Depuis PC3
ping 172.17.30.1
! → réussit ✅ (passerelle VLAN 30)

ping 172.17.10.10
! → réussit ✅ (PC1, via routage inter-VLAN)
```

**Réponse — Quelles adresses PC1 et PC3 utilisent comme passerelle ?**
- PC1 : `172.17.10.1` → sous-interface G0/0.10 de R1
- PC3 : `172.17.30.1` → sous-interface G0/0.30 de R1

---

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `interface g0/0.10` | Crée la sous-interface G0/0.10 |
| `encapsulation dot1Q <vlan>` | Associe la sous-interface à un VLAN (tagging 802.1Q) |
| `ip address <ip> <masque>` | Assigne l'IP de la sous-interface (= passerelle du VLAN) |
| `interface g0/0` + `no shutdown` | Active l'interface physique parente (active aussi les subifs) |
| `show ip interface brief` | Vérifie l'état des interfaces et sous-interfaces |
| `switchport mode trunk` | Active le trunk sur le port switch → routeur |
| `show interfaces trunk` | Vérifie les trunks actifs |
| `show vlan brief` | Vérifie l'assignation des ports aux VLANs |

---

> ⚠️ **À retenir — les 3 conditions pour que Router-on-a-Stick fonctionne** :
> 1. Les **VLANs** doivent exister sur le switch et les ports d'accès doivent être assignés.
> 2. Les **sous-interfaces** du routeur doivent avoir `encapsulation dot1Q` + une IP.
> 3. L'interface **physique** du routeur doit être `no shutdown`, et le port switch qui la connecte doit être en **trunk**.

---

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : VLANs sur S1 ─────────────────
! ══════════════════════════════════════════════

    ! ─── User EXEC ───────────────────────────
    enable

        ! ─── Privileged EXEC ─────────────────
        configure terminal

            ! ─── Global Config ───────────────
            vlan 10
                ! ─── VLAN Config ─────────────
                exit

            vlan 30
                ! ─── VLAN Config ─────────────
                exit

            interface f0/11
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                exit

            interface f0/6
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            end

        show vlan brief

! ─── Test initial (échoue) ────────────────────
! PC1 > ping 172.17.30.10  → échoue ✗


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Sous-interfaces sur R1 ────────
! ══════════════════════════════════════════════

    ! ─── User EXEC ───────────────────────────
    enable

        ! ─── Privileged EXEC ─────────────────
        show ip interface brief
        ! → G0/0.10 et G0/0.30 DOWN (G0/0 pas activée)

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/0.10
                ! ─── Subinterface Config ─────
                encapsulation dot1Q 10
                ip address 172.17.10.1 255.255.255.0
                exit

            interface g0/0.30
                ! ─── Subinterface Config ─────
                encapsulation dot1Q 30
                ip address 172.17.30.1 255.255.255.0
                exit

            interface g0/0
                ! ─── Interface Config ────────
                no shutdown
                exit

            end

        show ip interface brief
        ! → G0/0, G0/0.10, G0/0.30 UP/UP ✅


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Trunk et vérification ─────────
! ══════════════════════════════════════════════

! ─── Test intermédiaire (toujours en échec) ───
! PC1 > ping 172.17.30.10  → échoue ✗ (trunk manquant)

! ─── Sur S1 ───────────────────────────────────

    enable

        show vlan
        ! → G0/1 dans VLAN 1 par défaut

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show interfaces trunk
        ! → G0/1 en trunking ✅

        show vlan brief
        ! → G0/1 n'apparaît plus dans aucun VLAN (c'est un trunk)


! ─── Vérification finale ──────────────────────
! PC1 > ping 172.17.10.1   → réussit ✅ (gateway)
! PC1 > ping 172.17.30.10  → réussit ✅ (PC3, inter-VLAN)
! PC3 > ping 172.17.30.1   → réussit ✅ (gateway)
! PC3 > ping 172.17.10.10  → réussit ✅ (PC1, inter-VLAN)
```
