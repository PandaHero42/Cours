# Packet Tracer – Troubleshoot Inter-VLAN Routing

> **Objectif** : Identifier et corriger les erreurs de configuration d'un réseau Router-on-a-Stick qui empêchent la communication inter-VLAN entre PC1 et PC3.

---

## Topologie

| Device | Interface | Adresse IP   | Masque        | Passerelle  | VLAN    |
|--------|-----------|--------------|---------------|-------------|---------|
| R1     | G0/1.10   | 172.17.10.1  | 255.255.255.0 | N/A         | VLAN 10 |
| R1     | G0/1.30   | 172.17.30.1  | 255.255.255.0 | N/A         | VLAN 30 |
| PC1    | NIC       | 172.17.10.10 | 255.255.255.0 | 172.17.10.1 | VLAN 10 |
| PC3    | NIC       | 172.17.30.10 | 255.255.255.0 | 172.17.30.1 | VLAN 30 |

---

## Partie 1 : Localiser les problèmes

### Méthodologie de dépannage

Partir du bas de la pile (physique → logique) et remonter :

```
1. L'interface physique est-elle active ?        → show ip interface brief
2. Les sous-interfaces ont-elles le bon VLAN ?   → show interface g0/1.10
3. Les IPs sont-elles correctes ?                → show interface g0/1.10 / g0/1.30
4. Le port switch est-il bien en trunk ?         → show interfaces trunk (sur S1)
5. Les VLANs existent-ils sur le switch ?        → show vlan brief (sur S1)
6. Les ports PC sont-ils dans le bon VLAN ?      → show vlan brief (sur S1)
```

### Commandes de diagnostic sur R1

```
R1# show ip interface brief
! → vérifier que G0/1, G0/1.10 et G0/1.30 sont UP/UP
! → vérifier les IPs assignées

R1# show interface g0/1.10
! → vérifier : encapsulation 802.1Q VLAN 10, IP 172.17.10.1

R1# show interface g0/1.30
! → vérifier : encapsulation 802.1Q VLAN 30, IP 172.17.30.1
```

### Commandes de diagnostic sur S1

```
S1# show interfaces trunk
! → vérifier que le port vers R1 est bien en mode trunk
! → vérifier les VLANs autorisés sur le trunk

S1# show vlan brief
! → vérifier que VLAN 10 et VLAN 30 existent
! → vérifier que F0/6 (PC3) est dans VLAN 30
! → vérifier que F0/11 (PC1) est dans VLAN 10
```

## Problèmes courants et solutions

> ⚠️ Les erreurs exactes dépendent du fichier Packet Tracer. Voici tous les problèmes classiques de Router-on-a-Stick avec leurs symptômes et corrections.

### Tableau de documentation

| Problème | Symptôme | Solution |
|----------|----------|----------|
| Interface physique G0/1 en shutdown | G0/1.10 et G0/1.30 DOWN | `interface g0/1` + `no shutdown` |
| Mauvais VLAN dans `encapsulation dot1Q` | Sous-interface associée au mauvais VLAN | Reconfigurer `encapsulation dot1Q <bon_vlan>` |
| IP incorrecte sur une sous-interface | Ping vers la gateway échoue | `ip address <bonne_ip> <masque>` |
| Port trunk non configuré sur S1 | Tout trafic VLAN bloqué inter-switch | `switchport mode trunk` sur port vers R1 |
| Port PC dans le mauvais VLAN | PC ne peut pinguer que certains équipements | `switchport access vlan <bon_vlan>` |
| VLAN inexistant sur S1 | Port assigné à un VLAN qui n'existe pas | `vlan <num>` pour créer le VLAN |
| Mauvaise IP ou gateway sur PC | Ping vers gateway échoue depuis le PC | Corriger via Desktop > IP Configuration |

## Partie 2 : Appliquer les corrections

### Correction type — Interface physique inactive

```
R1> enable
R1# configure terminal

R1(config)# interface g0/1
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# end
```

### Correction type — Mauvais VLAN sur sous-interface

Si les VLANs sont simplement inversés (g0/1.10 configuré avec VLAN 30 et g0/1.30 avec VLAN 10), on ne peut pas les échanger directement — Cisco refuse d'assigner un VLAN déjà utilisé sur une autre sous-interface de la même interface physique.

La solution est d'utiliser un **VLAN temporaire** comme tampon pour libérer le conflit :

```
R1# configure terminal

R1(config)# interface g0/1.10
R1(config-subif)# encapsulation dot1Q 99
! → VLAN temporaire : libère VLAN 30

R1(config)# interface g0/1.30
R1(config-subif)# encapsulation dot1Q 30
! → VLAN 30 maintenant libre, on l'assigne correctement

R1(config)# interface g0/1.10
R1(config-subif)# encapsulation dot1Q 10
! → VLAN 10 maintenant libre, on l'assigne correctement

R1(config)# end
```

Si les IPs doivent aussi être corrigées :

```
R1# configure terminal

R1(config)# interface g0/1.10
R1(config-subif)# ip address 172.17.10.1 255.255.255.0
R1(config-subif)# exit

R1(config)# interface g0/1.30
R1(config-subif)# ip address 172.17.30.1 255.255.255.0
R1(config-subif)# exit

R1(config)# end
```

### Correction type — Port trunk manquant sur S1

```
S1> enable
S1# configure terminal

S1(config)# interface g0/1
S1(config-if)# switchport mode trunk
S1(config-if)# exit

S1(config)# end
```

### Correction type — Port PC dans le mauvais VLAN

```
S1# configure terminal

S1(config)# interface f0/11
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 10
S1(config-if)# exit

S1(config)# interface f0/6
S1(config-if)# switchport mode access
S1(config-if)# switchport access vlan 30
S1(config-if)# exit

S1(config)# end
```

### Correction type — VLAN manquant sur S1

```
S1# configure terminal

S1(config)# vlan 10
S1(config-vlan)# exit

S1(config)# vlan 30
S1(config-vlan)# exit

S1(config)# end
```

## Partie 3 : Vérifier la connectivité

```
! Depuis PC1
ping 172.17.10.1
! → réussit ✅ (gateway VLAN 10)

ping 172.17.30.10
! → réussit ✅ (PC3, inter-VLAN via R1)

! Depuis PC3
ping 172.17.30.1
! → réussit ✅ (gateway VLAN 30)

ping 172.17.10.10
! → réussit ✅ (PC1, inter-VLAN via R1)
```

## Résumé des commandes de diagnostic

| Commande | Ce qu'elle vérifie |
|----------|--------------------|
| `show ip interface brief` | État UP/DOWN et IPs de toutes les interfaces |
| `show interface g0/1.10` | Détails sous-interface : encapsulation, VLAN, IP |
| `show interface g0/1.30` | Idem pour VLAN 30 |
| `show interfaces trunk` | Ports trunk actifs et VLANs autorisés (sur S1) |
| `show vlan brief` | VLANs existants et ports associés (sur S1) |
| `show running-config` | Configuration complète pour comparaison |

---

> 💡 **Approche recommandée** : tester la connectivité étape par étape en partant des éléments les plus proches.
> 1. PC1 → gateway `172.17.10.1` (teste VLAN 10 local + sous-interface R1)
> 2. PC3 → gateway `172.17.30.1` (teste VLAN 30 local + sous-interface R1)
> 3. PC1 → PC3 (teste le routage inter-VLAN complet)
>
> Si l'étape 1 échoue, le problème est local au VLAN ou à la sous-interface.  
> Si les étapes 1 et 2 réussissent mais pas 3, le problème est dans le routage R1.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Diagnostic ────────────────────
! ══════════════════════════════════════════════

! ─── Sur R1 ───────────────────────────────────

    enable

        show ip interface brief
        ! → vérifier UP/UP sur G0/1, G0/1.10, G0/1.30

        show interface g0/1.10
        ! → vérifier encapsulation dot1Q VLAN 10
        ! → vérifier IP 172.17.10.1/24

        show interface g0/1.30
        ! → vérifier encapsulation dot1Q VLAN 30
        ! → vérifier IP 172.17.30.1/24


! ─── Sur S1 ───────────────────────────────────

    enable

        show interfaces trunk
        ! → vérifier port trunk vers R1 actif

        show vlan brief
        ! → vérifier VLAN 10 et 30 existent
        ! → vérifier F0/11 dans VLAN 10, F0/6 dans VLAN 30


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Corrections ───────────────────
! ══════════════════════════════════════════════

! ─── Sur R1 (appliquer selon les erreurs trouvées) ─

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                no shutdown
                exit

            interface g0/1.10
                ! ─── Subinterface Config ─────
                encapsulation dot1Q 99
                ! → VLAN temporaire pour libérer le conflit
                exit

            interface g0/1.30
                ! ─── Subinterface Config ─────
                encapsulation dot1Q 30
                ip address 172.17.30.1 255.255.255.0
                exit

            interface g0/1.10
                ! ─── Subinterface Config ─────
                encapsulation dot1Q 10
                ip address 172.17.10.1 255.255.255.0
                exit

            end


! ─── Sur S1 (appliquer selon les erreurs trouvées) ─

    enable

        configure terminal

            ! ─── Global Config ───────────────
            vlan 10
                ! ─── VLAN Config ─────────────
                exit

            vlan 30
                ! ─── VLAN Config ─────────────
                exit

            interface g0/1
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            interface f0/11
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                exit

            interface f0/6
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            end


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Vérification finale ───────────
! ══════════════════════════════════════════════

! PC1 > ping 172.17.10.1   → ✅ (gateway VLAN 10)
! PC1 > ping 172.17.30.10  → ✅ (PC3, inter-VLAN)
! PC3 > ping 172.17.30.1   → ✅ (gateway VLAN 30)
! PC3 > ping 172.17.10.10  → ✅ (PC1, inter-VLAN)
```
