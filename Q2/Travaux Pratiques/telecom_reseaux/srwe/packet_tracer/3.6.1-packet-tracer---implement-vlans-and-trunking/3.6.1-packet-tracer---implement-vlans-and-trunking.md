# Packet Tracer – Implement VLANs and Trunking

> **Objectif** : Exercice de synthèse — configurer les VLANs, assigner les ports, configurer les interfaces de gestion SVI, et mettre en place les trunks statiques et dynamiques entre trois switches.

---

## Topologie

| Device | Interface | Adresse IP     | Masque        | Port Switch | VLAN           |
|--------|-----------|----------------|---------------|-------------|----------------|
| PC1    | NIC       | 192.168.10.10  | 255.255.255.0 | SWB F0/1    | 10             |
| PC2    | NIC       | 192.168.20.20  | 255.255.255.0 | SWB F0/2    | 20             |
| PC3    | NIC       | 192.168.30.30  | 255.255.255.0 | SWB F0/3    | 30             |
| PC4    | NIC       | 192.168.10.11  | 255.255.255.0 | SWC F0/1    | 10             |
| PC5    | NIC       | 192.168.20.21  | 255.255.255.0 | SWC F0/2    | 20             |
| PC6    | NIC       | 192.168.30.31  | 255.255.255.0 | SWC F0/3    | 30             |
| PC7    | NIC       | 192.168.10.12  | 255.255.255.0 | SWC F0/4    | 10 + 40 (Voice)|
| SWA    | SVI       | 192.168.99.252 | 255.255.255.0 | N/A         | 99             |
| SWB    | SVI       | 192.168.99.253 | 255.255.255.0 | N/A         | 99             |
| SWC    | SVI       | 192.168.99.254 | 255.255.255.0 | N/A         | 99             |

### VLANs à créer (sur les 3 switches)

| VLAN | Nom        |
|------|------------|
| 10   | Admin      |
| 20   | Accounts   |
| 30   | HR         |
| 40   | Voice      |
| 99   | Management |
| 100  | Native     |

### Liens trunk

| Lien       | Type    | Mode SWA             | Mode SWB/SWC       |
|------------|---------|----------------------|--------------------|
| SWA ↔ SWB | Statique| trunk + nonegotiate  | trunk + nonegotiate|
| SWA ↔ SWC | Dynamique| dynamic desirable   | dynamic auto (défaut)|

> VLAN natif = **VLAN 100** sur tous les trunks.

## Partie 1 : Configurer les VLANs

> ⚠️ Les noms doivent correspondre **exactement** au tableau — sensibles à la casse.

Les mêmes VLANs doivent être créés sur **SWA, SWB et SWC**.

**Sur SWA (répéter identique sur SWB et SWC) :**

```
SWA> enable
SWA# configure terminal

SWA(config)# vlan 10
SWA(config-vlan)# name Admin
SWA(config-vlan)# exit

SWA(config)# vlan 20
SWA(config-vlan)# name Accounts
SWA(config-vlan)# exit

SWA(config)# vlan 30
SWA(config-vlan)# name HR
SWA(config-vlan)# exit

SWA(config)# vlan 40
SWA(config-vlan)# name Voice
SWA(config-vlan)# exit

SWA(config-vlan)# exit

SWA(config)# vlan 100
SWA(config-vlan)# name Native
SWA(config-vlan)# exit

```

### Étape 3 – Port voix sur SWC F0/4 (PC7 + IP Phone)

F0/4 gère simultanément les données de PC7 (VLAN 10) et la voix IP (VLAN 40).  
`mls qos trust cos` fait confiance aux valeurs QoS du téléphone pour prioriser la voix.

```
SWC(config)# interface f0/4
SWC(config-if)# switchport mode access
SWC(config-if)# switchport access vlan 10
SWC(config-if)# mls qos trust cos
SWC(config-if)# switchport voice vlan 40
SWC(config-if)# exit

SWC(config)# end
```

### Étape 4 – Configurer les interfaces de gestion SVI

L'interface **VLAN 99** est l'interface virtuelle de gestion du switch (SVI — Switched Virtual Interface).  
Elle permet d'administrer le switch à distance via Telnet/SSH.  

> ⚠️ L'énoncé précise que les switches **ne doivent pas pouvoir se pinguer** — les SVIs sont configurées mais aucune passerelle par défaut n'est définie, et les trunks ne sont pas encore actifs à cette étape.

## Partie 3 : Configurer le trunk statique (SWA ↔ SWB)

Le lien entre SWA et SWB est configuré en **trunk statique** des deux côtés, avec **DTP désactivé** (`nonegotiate`).  
Le VLAN natif est mis à **VLAN 100** pour éviter d'utiliser le VLAN 1 par défaut.

**Sur SWA :**

```
SWA> enable
SWA# configure terminal

SWA(config)# interface g0/1
SWA(config-if)# switchport mode trunk
SWA(config-if)# switchport nonegotiate
SWA(config-if)# switchport trunk native vlan 100
SWA(config-if)# exit

SWA(config)# end
SWA# show interfaces trunk
```

**Sur SWB :**

```
SWB> enable
SWB# configure terminal

SWB(config)# interface g0/1
SWB(config-if)# switchport mode trunk
SWB(config-if)# switchport nonegotiate
SWB(config-if)# switchport trunk native vlan 100
SWB(config-if)# exit

SWB(config)# end
SWB# show interfaces trunk
```

Les deux côtés sont en `trunk + nonegotiate` → aucun paquet DTP envoyé, trunk statique actif ✅.

---

## Partie 4 : Configurer le trunk dynamique (SWA ↔ SWC)

SWC est un 2960 — son mode DTP par défaut est `dynamic auto` (attend passivement).  
Pour que le trunk s'établisse, SWA doit initier avec `dynamic desirable`.

**Sur SWA :**

```
SWA> enable
SWA# configure terminal

SWA(config)# interface g0/2
SWA(config-if)# switchport mode dynamic desirable
SWA(config-if)# switchport trunk native vlan 100
SWA(config-if)# exit

SWA(config)# end
```

**Sur SWC** — configurer le VLAN natif (le trunk est déjà négocié automatiquement grâce à DTP) :

```
SWC> enable
SWC# configure terminal

SWC(config)# interface g0/1
SWC(config-if)# switchport trunk native vlan 100
SWC(config-if)# exit

SWC(config)# end
```

Vérifier que les trunks sont actifs sur tous les switches :

```
SWA# show interfaces trunk
SWB# show interfaces trunk
SWC# show interfaces trunk
```

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `vlan <num>` + `name <nom>` | Créer et nommer un VLAN |
| `switchport mode access` | Port d'accès (1 seul VLAN) |
| `switchport access vlan <num>` | Assigner le port à un VLAN |
| `switchport voice vlan <num>` | VLAN voix pour téléphonie IP |
| `mls qos trust cos` | Activer la priorité QoS pour la voix |
| `interface vlan <num>` | Interface SVI de gestion du switch |
| `switchport mode trunk` | Trunk statique forcé |
| `switchport nonegotiate` | Désactiver DTP |
| `switchport mode dynamic desirable` | Initier la négociation trunk (DTP) |
| `switchport trunk native vlan <num>` | Définir le VLAN natif du trunk |
| `show interfaces trunk` | Vérifier les trunks actifs |
| `show vlan brief` | Vérifier VLANs et ports |

---

> ⚠️ **À retenir** :
> - `nonegotiate` des **deux côtés** = trunk statique sans DTP.
> - `desirable` + `auto` (défaut 2960) = trunk dynamique négocié.
> - Le VLAN natif doit être **identique des deux côtés** du trunk.
> - Les VLANs doivent être créés sur **tous les switches** traversés par le trunk.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Création VLANs ────────────────
! (Sur SWA, SWB et SWC — répéter sur chacun)
! ══════════════════════════════════════════════

    enable

        configure terminal

            ! ─── Global Config ───────────────
            vlan 10
                ! ─── VLAN Config ─────────────
                name Admin
                exit

            vlan 20
                ! ─── VLAN Config ─────────────
                name Accounts
                exit

            vlan 30
                ! ─── VLAN Config ─────────────
                name HR
                exit

            vlan 40
                ! ─── VLAN Config ─────────────
                name Voice
                exit

            vlan 99
                ! ─── VLAN Config ─────────────
                name Management
                exit

            vlan 100
                ! ─── VLAN Config ─────────────
                name Native
                exit

            end

        show vlan brief


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Assignation des ports ─────────
! ══════════════════════════════════════════════

! ─── Sur SWB ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface f0/1
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                exit

            interface f0/2
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 20
                exit

            interface f0/3
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            interface vlan 99
                ! ─── Interface Config ────────
                ip address 192.168.99.253 255.255.255.0
                no shutdown
                exit

            end


! ─── Sur SWC ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface f0/1
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                exit

            interface f0/2
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 20
                exit

            interface f0/3
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            interface f0/4
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                mls qos trust cos
                switchport voice vlan 40
                ! → données PC7 = VLAN 10, voix = VLAN 40
                exit

            interface vlan 99
                ! ─── Interface Config ────────
                ip address 192.168.99.254 255.255.255.0
                no shutdown
                exit

            end


! ─── Sur SWA ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface vlan 99
                ! ─── Interface Config ────────
                ip address 192.168.99.252 255.255.255.0
                no shutdown
                exit

            end


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Trunk statique SWA ↔ SWB ──────
! ══════════════════════════════════════════════

! ─── Sur SWA ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport mode trunk
                switchport nonegotiate
                switchport trunk native vlan 100
                exit

            end

        show interfaces trunk


! ─── Sur SWB ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport mode trunk
                switchport nonegotiate
                switchport trunk native vlan 100
                exit

            end

        show interfaces trunk


! ══════════════════════════════════════════════
! ─── PARTIE 4 : Trunk dynamique SWA ↔ SWC ─────
! ══════════════════════════════════════════════

! ─── Sur SWA ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/2
                ! ─── Interface Config ────────
                switchport mode dynamic desirable
                ! → SWC en auto (défaut) accepte → trunk négocié
                switchport trunk native vlan 100
                exit

            end

        show interfaces trunk


! ─── Sur SWC ──────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport trunk native vlan 100
                exit

            end

        show interfaces trunk


! ══════════════════════════════════════════════
! ─── Vérification finale ──────────────────────
! ══════════════════════════════════════════════

! PC1 > ping 192.168.10.11  → réussit ✅ (PC4, VLAN 10)
! PC2 > ping 192.168.20.21  → réussit ✅ (PC5, VLAN 20)
! PC3 > ping 192.168.30.31  → réussit ✅ (PC6, VLAN 30)
```
