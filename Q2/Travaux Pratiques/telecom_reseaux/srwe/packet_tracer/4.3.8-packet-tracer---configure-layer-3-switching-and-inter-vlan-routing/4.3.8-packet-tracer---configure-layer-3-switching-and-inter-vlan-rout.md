# Packet Tracer – Configure Layer 3 Switching and Inter-VLAN Routing

> **Objectif** : Configurer un switch multicouche (Layer 3) pour faire du routage inter-VLAN directement sans routeur externe, en IPv4 et IPv6. Le switch MLS remplace à la fois le switch et le routeur.

### Concept : Layer 3 Switch vs Router-on-a-Stick

```
Router-on-a-Stick :   Switch L2 ──[trunk]── Routeur (subinterfaces)
Layer 3 Switch    :   Switch L3 (MLS) — fait les deux en interne
                       ├── SVIs VLAN 10/20/30/99 = passerelles
                       └── G0/2 = port routé vers l'extérieur (Cloud)
```

> 💡 Un **switch multicouche (MLS)** peut router entre VLANs via des **SVI (Switched Virtual Interfaces)** sans avoir besoin d'un routeur externe. La commande `ip routing` active cette capacité. Un port physique peut aussi être converti en port routé (Layer 3) avec `no switchport`.

---

## Partie 1 : Configurer le Layer 3 Switching

### Convertir G0/2 en port routé

Par défaut, tous les ports d'un switch sont des ports de couche 2 (switching).  
`no switchport` convertit le port en interface routée de couche 3 — il peut alors recevoir une adresse IP et router des paquets comme un port de routeur.

```
MLS> enable
MLS# configure terminal

MLS(config)# interface g0/2
MLS(config-if)# no switchport
MLS(config-if)# ip address 209.165.200.225 255.255.255.252
MLS(config-if)# no shutdown
MLS(config-if)# exit

MLS(config)# end
```

### Vérifier la connectivité vers le Cloud

```
MLS# ping 209.165.200.226
! → 5/5 succès ✅
```

## Partie 2 : Configurer le routage inter-VLAN (IPv4)

### Étape 1 – Créer les VLANs sur MLS

> ⚠️ Les noms sont **sensibles à la casse** — respecter exactement `Staff`, `Student`, `Faculty`.

```
MLS# configure terminal

MLS(config)# vlan 10
MLS(config-vlan)# name Staff
MLS(config-vlan)# exit

MLS(config)# vlan 20
MLS(config-vlan)# name Student
MLS(config-vlan)# exit

MLS(config)# vlan 30
MLS(config-vlan)# name Faculty
MLS(config-vlan)# exit

MLS(config)# end
```

### Étape 2 – Configurer les SVIs sur MLS

Les **SVI (Switched Virtual Interfaces)** sont les interfaces virtuelles associées à chaque VLAN sur le switch L3.  
Elles servent de **passerelle par défaut** pour les hôtes de chaque VLAN — exactement comme les sous-interfaces dans Router-on-a-Stick, mais sans routeur externe.

```
MLS# configure terminal

MLS(config)# interface vlan 10
MLS(config-if)# ip address 192.168.10.254 255.255.255.0
MLS(config-if)# no shutdown
MLS(config-if)# exit

MLS(config)# interface vlan 20
MLS(config-if)# ip address 192.168.20.254 255.255.255.0
MLS(config-if)# no shutdown
MLS(config-if)# exit

MLS(config)# interface vlan 30
MLS(config-if)# ip address 192.168.30.254 255.255.255.0
MLS(config-if)# no shutdown
MLS(config-if)# exit

MLS(config)# interface vlan 99
MLS(config-if)# ip address 192.168.99.254 255.255.255.0
MLS(config-if)# no shutdown
MLS(config-if)# exit

MLS(config)# end
```

### Étape 3 – Configurer le trunk sur MLS G0/1

Sur un switch L3, la configuration trunk inclut une étape supplémentaire : `switchport trunk encapsulation dot1q` doit être spécifié **avant** `switchport mode trunk` (contrairement à un switch L2 où c'est implicite).

```
MLS# configure terminal

MLS(config)# interface g0/1
MLS(config-if)# switchport trunk encapsulation dot1q
MLS(config-if)# switchport mode trunk
MLS(config-if)# switchport trunk native vlan 99
MLS(config-if)# exit

MLS(config)# end
```

### Étape 4 – Configurer le trunk sur S1 G0/1

```
S1> enable
S1# configure terminal

S1(config)# interface g0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk native vlan 99
S1(config-if)# exit

S1(config)# end
```

### Étape 5 – Activer le routage IP sur MLS

> ⚠️ Sans `ip routing`, le switch L3 ne route pas — les SVIs sont configurées mais le switch se comporte comme un switch L2 ordinaire. C'est la commande clé qui active la fonctionnalité de routage.

```
MLS# show ip route
! → aucune route active (routing désactivé)

MLS# configure terminal

MLS(config)# ip routing

MLS(config)# end

MLS# show ip route
! → toutes les routes directement connectées apparaissent :
! C 192.168.10.0/24 via Vlan10
! C 192.168.20.0/24 via Vlan20
! C 192.168.30.0/24 via Vlan30
! C 192.168.99.0/24 via Vlan99
! C 209.165.200.224/30 via GigabitEthernet0/2
```

### Étape 6 – Vérifier la connectivité IPv4

```
! Intra-VLAN 10 :
! PC0 > ping 192.168.10.2   → réussit ✅ (PC3)
! PC0 > ping 192.168.10.254 → réussit ✅ (MLS SVI)

! Intra-VLAN 20 :
! PC1 > ping 192.168.20.2   → réussit ✅ (PC4)

! Intra-VLAN 30 :
! PC2 > ping 192.168.30.2   → réussit ✅ (PC5)

! Inter-VLAN :
! PC0 > ping 192.168.20.1   → réussit ✅ (PC1, VLAN 20)
! PC0 > ping 192.168.30.1   → réussit ✅ (PC2, VLAN 30)

! Vers Cloud :
! PC0 > ping 209.165.200.226 → réussit ✅

! Gestion switches :
! S1 > ping 192.168.99.2    → réussit ✅ (S2)
! S1 > ping 192.168.99.254  → réussit ✅ (MLS)
```

## Partie 3 : Configurer le routage inter-VLAN IPv6

### Étape 1 – Activer le routage IPv6

```
MLS# configure terminal

MLS(config)# ipv6 unicast-routing

MLS(config)# end
```

### Étape 2 – Configurer les adresses IPv6 sur les SVIs

On ajoute les adresses IPv6 aux interfaces VLAN déjà créées — elles auront alors à la fois une IPv4 et une IPv6.

```
MLS# configure terminal

MLS(config)# interface vlan 10
MLS(config-if)# ipv6 address 2001:db8:acad:10::1/64
MLS(config-if)# exit

MLS(config)# interface vlan 20
MLS(config-if)# ipv6 address 2001:db8:acad:20::1/64
MLS(config-if)# exit

MLS(config)# interface vlan 30
MLS(config-if)# ipv6 address 2001:db8:acad:30::1/64
MLS(config-if)# exit

MLS(config)# end
```

### Étape 3 – Configurer l'IPv6 sur G0/2 (vers Cloud)

```
MLS# configure terminal

MLS(config)# interface g0/2
MLS(config-if)# ipv6 address 2001:db8:acad:a::1/64
MLS(config-if)# exit

MLS(config)# end

MLS# show ipv6 route
! → routes C connectées pour chaque réseau IPv6
```

### Étape 4 – Vérifier la connectivité IPv6

```
! PC4 > ping 2001:db8:acad:20::1  → réussit ✅ (MLS, VLAN 20)
```

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `no switchport` | Convertit un port L2 en port routé L3 |
| `ip routing` | Active le routage IPv4 sur le switch L3 |
| `ipv6 unicast-routing` | Active le routage IPv6 sur le switch L3 |
| `interface vlan <num>` | Crée/accède à la SVI d'un VLAN |
| `switchport trunk encapsulation dot1q` | Spécifie l'encapsulation trunk (requis sur L3 switch) |
| `switchport mode trunk` | Active le mode trunk |
| `switchport trunk native vlan <num>` | Définit le VLAN natif |
| `show ip route` | Affiche la table de routage IPv4 |
| `show ipv6 route` | Affiche la table de routage IPv6 |
| `show ip interface brief` | Vérifie l'état de toutes les interfaces |

---

> ⚠️ **Différences clés entre Router-on-a-Stick et Layer 3 Switch** :
>
> | | Router-on-a-Stick | Layer 3 Switch (MLS) |
> |--|-------------------|----------------------|
> | Matériel | Switch L2 + routeur séparé | Un seul switch L3 |
> | Passerelles | Sous-interfaces routeur | SVIs sur le switch |
> | Activation routing | Toujours actif | `ip routing` requis |
> | Trunk encapsulation | `encapsulation dot1Q` sur subif | `switchport trunk encapsulation dot1q` sur port |
> | Performance | Limité par le lien trunk | Routing interne (plus rapide) |

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Port routé G0/2 ───────────────
! ══════════════════════════════════════════════

    ! ─── User EXEC ───────────────────────────
    enable

        ! ─── Privileged EXEC ─────────────────
        configure terminal

            ! ─── Global Config ───────────────
            interface g0/2
                ! ─── Interface Config ────────
                no switchport
                ip address 209.165.200.225 255.255.255.252
                no shutdown
                exit

            end

        ping 209.165.200.226
        ! → 5/5 ✅


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Routage inter-VLAN IPv4 ───────
! ══════════════════════════════════════════════

        configure terminal

            ! ─── Global Config ───────────────
            vlan 10
                ! ─── VLAN Config ─────────────
                name Staff
                exit

            vlan 20
                ! ─── VLAN Config ─────────────
                name Student
                exit

            vlan 30
                ! ─── VLAN Config ─────────────
                name Faculty
                exit

            interface vlan 10
                ! ─── Interface Config ────────
                ip address 192.168.10.254 255.255.255.0
                no shutdown
                exit

            interface vlan 20
                ! ─── Interface Config ────────
                ip address 192.168.20.254 255.255.255.0
                no shutdown
                exit

            interface vlan 30
                ! ─── Interface Config ────────
                ip address 192.168.30.254 255.255.255.0
                no shutdown
                exit

            interface vlan 99
                ! ─── Interface Config ────────
                ip address 192.168.99.254 255.255.255.0
                no shutdown
                exit

            interface g0/1
                ! ─── Interface Config ────────
                switchport trunk encapsulation dot1q
                switchport mode trunk
                switchport trunk native vlan 99
                exit

            ip routing
            ! → active le routage IPv4 inter-VLAN

            end

        show ip route
        ! → routes C pour chaque VLAN apparaissent ✅


! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport mode trunk
                switchport trunk native vlan 99
                exit

            end


! ─── Vérifications IPv4 ───────────────────────
! PC0 > ping 192.168.10.254  → ✅ (MLS VLAN 10)
! PC0 > ping 192.168.20.1    → ✅ (PC1, inter-VLAN)
! PC0 > ping 209.165.200.226 → ✅ (Cloud)
! S1  > ping 192.168.99.254  → ✅ (MLS VLAN 99)


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Routage inter-VLAN IPv6 ───────
! ══════════════════════════════════════════════

    enable

        configure terminal

            ! ─── Global Config ───────────────
            ipv6 unicast-routing

            interface vlan 10
                ! ─── Interface Config ────────
                ipv6 address 2001:db8:acad:10::1/64
                exit

            interface vlan 20
                ! ─── Interface Config ────────
                ipv6 address 2001:db8:acad:20::1/64
                exit

            interface vlan 30
                ! ─── Interface Config ────────
                ipv6 address 2001:db8:acad:30::1/64
                exit

            interface g0/2
                ! ─── Interface Config ────────
                ipv6 address 2001:db8:acad:a::1/64
                exit

            end

        show ipv6 route
        ! → routes C pour chaque réseau IPv6 ✅


! ─── Vérifications IPv6 ───────────────────────
! PC3 > ping 2001:db8:acad:10::1  → ✅ (MLS VLAN 10)
! PC3 > ping 2001:db8:acad:20::2  → ✅ (PC4, inter-VLAN)
```
