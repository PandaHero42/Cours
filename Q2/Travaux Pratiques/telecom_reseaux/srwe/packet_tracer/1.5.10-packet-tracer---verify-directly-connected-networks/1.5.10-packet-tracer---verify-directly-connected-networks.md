# Packet Tracer – Verify Directly Connected Networks

> **Objectif** : Vérifier et corriger l'adressage IPv4 sur R1 et IPv6 sur R2, puis confirmer la connectivité entre les LANs. C'est un exercice de **vérification et dépannage**, pas de configuration from scratch.

---

|--------|-----------|---------------------------|-----------------------|

## Partie 1 : Vérifier les réseaux directement connectés en IPv4 (R1)

### Étape 1a – Vérifier le statut des interfaces

On affiche uniquement les interfaces qui ont une IP assignée, en excluant les lignes "unassigned".  
Le filtre `| exclude unassigned` masque les interfaces non configurées pour une lecture plus claire.

```
R1# show ip interface brief | exclude unassigned
```

Résultat attendu : les interfaces G0/0/0, G0/0/1 et S0/1/0 doivent apparaître avec le statut `up/up`.  
Si une interface est en `administratively down`, il faut la réactiver avec `no shutdown`.

**Correction d'une interface désactivée :**

```
R1# configure terminal
R1(config)# interface g0/0/0
R1(config-if)# no shutdown
R1(config-if)# exit
R1(config)# interface g0/0/1
R1(config-if)# no shutdown
R1(config-if)# exit
```

**Correction d'une adresse IP erronée** (si l'IP ne correspond pas au tableau) :

```
R1(config)# interface g0/0/0
R1(config-if)# ip address 172.16.20.1 255.255.255.128
R1(config-if)# exit
R1(config)# interface g0/0/1
R1(config-if)# ip address 172.16.20.129 255.255.255.128
R1(config-if)# exit
R1(config)# end
```

### Étape 1b – Afficher la table de routage

On filtre la table de routage pour commencer l'affichage à partir du mot "Gateway".  
Le filtre `| begin` affiche tout à partir de la première ligne qui contient le mot clé.

> 💡 Le filtre est **sensible à la casse**. `Gate`, `Gatew`, `Gateway` fonctionnent tous. `G` seul ne fonctionne pas car trop ambigu.

```
R1# show ip route | begin Gate
```

La **Gateway of last resort** est la route par défaut (0.0.0.0/0) — c'est vers elle que le routeur envoie les paquets dont il ne connaît pas la destination.

### Étape 1c – Afficher les descriptions et connexions des interfaces

On peut combiner deux critères de recherche avec le symbole `|` dans le filtre.  
Cette commande affiche toutes les lignes contenant "Desc" ou "conn".

```
R1# show interface | include Desc|conn
```

### Étape 1d – Afficher les détails de G0/0/0 (duplex, vitesse, média)

On filtre les informations d'une interface spécifique pour voir ses paramètres physiques.

```
R1# show interface g0/0/0 | include duplex
```

### Étape 1e – Vérifier la connectivité IPv4

Depuis **PC1** (Desktop → Command Prompt) :

```
ping 172.16.20.138
```

Depuis **PC2** :

```
ping 172.16.20.10
```

Les pings doivent réussir ✅. Si ce n'est pas le cas, revérifier les IPs et le statut des interfaces.

## Partie 2 : Vérifier les réseaux directement connectés en IPv6 (R2)

### Étape 2a – Vérifier le statut des interfaces IPv6

```
R2# show ipv6 interface brief
```

Résultat attendu : G0/0/0, G0/0/1 et S0/1/1 en `up/up` avec les bonnes adresses.

### Étape 2b – Corriger une adresse IPv6 erronée

> ⚠️ En IPv6, une interface peut avoir **plusieurs adresses** simultanément. Il faut donc d'abord **supprimer** l'adresse incorrecte avant d'en ajouter une nouvelle, sinon les deux coexisteront.

Si G0/0/1 a une mauvaise adresse (ex: `2001:db8:c0de:14::1/64` au lieu de `2001:db8:c0de:13::1/64`) :

```
R2# configure terminal
R2(config)# interface g0/0/1
R2(config-if)# no ipv6 address 2001:db8:c0de:14::1/64
R2(config-if)# ipv6 address 2001:db8:c0de:13::1/64
R2(config-if)# exit
R2(config)# end
```

### Étape 2c – Afficher la table de routage IPv6

> 💡 Les filtres (`| begin`, `| include`, etc.) **ne fonctionnent pas** avec les commandes IPv6 dans Packet Tracer. On affiche la table complète.

```
R2# show ipv6 route
```

### Étape 2d – Afficher toutes les adresses IPv6 configurées via la running-config

Cette commande filtre la config en cours pour afficher uniquement les lignes contenant "ipv6" ou "interface", ce qui permet de voir d'un coup toutes les adresses IPv6 et sur quelles interfaces elles sont.

```
R2# show running-config | include ipv6|interface
```

Chaque interface Gigabit devrait avoir **2 adresses** : une globale unicast (`2001:...`) et une link-local (`fe80::...`).

### Étape 2e – Vérifier la connectivité IPv6

Depuis **PC3** (Desktop → Command Prompt) :

```
ping 2001:db8:c0de:13::a
```

Depuis **PC4** :

```
ping 2001:db8:c0de:12::a
```

Les pings doivent réussir ✅

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show ip interface brief \| exclude unassigned` | Affiche uniquement les interfaces avec une IP |
| `show ip route \| begin Gate` | Affiche la table de routage depuis "Gateway" |
| `show interface \| include Desc\|conn` | Affiche les lignes contenant description ou connected |
| `show interface g0/0/0 \| include duplex` | Affiche duplex/vitesse/média d'une interface |
| `show ipv6 interface brief` | Affiche l'état et les adresses IPv6 |
| `show ipv6 route` | Affiche la table de routage IPv6 |
| `show running-config \| include ipv6\|interface` | Affiche toutes les configs IPv6 de la running-config |
| `no ipv6 address <adresse>` | Supprime une adresse IPv6 spécifique d'une interface |

---

> ⚠️ **Rappel filtres** : Les filtres `| begin`, `| include`, `| exclude` sont **sensibles à la casse** et ne fonctionnent **pas** avec les commandes `show ipv6` dans Packet Tracer.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════════
! ─── Routeur R1 (vérification et correction IPv4) ─
! ══════════════════════════════════════════════════

    ! ─── User EXEC (mot de passe : cisco) ───────
    enable

        ! ─── Privileged EXEC (mdp : class) ──────
        show ip interface brief | exclude unassigned
        ! → vérifier statut up/up et IPs correctes

        configure terminal

            ! ─── Global Config ──────────────────
            interface g0/0/0
                ! ─── Interface Config ───────────
                ip address 172.16.20.1 255.255.255.128
                no shutdown
                exit

            interface g0/0/1
                ! ─── Interface Config ───────────
                ip address 172.16.20.129 255.255.255.128
                no shutdown
                exit

            end

        show ip route | begin Gate
        ! → noter la Gateway of last resort

        show interface | include Desc|conn
        ! → noter le Circuit ID affiché

        show interface g0/0/0 | include duplex
        ! → noter duplex, vitesse, type de média

! ─── Vérification connectivité depuis PC1 ─────────
ping 172.16.20.138

! ─── Vérification connectivité depuis PC2 ─────────
ping 172.16.20.10


! ══════════════════════════════════════════════════
! ─── Routeur R2 (vérification et correction IPv6) ─
! ══════════════════════════════════════════════════

    ! ─── User EXEC (mot de passe : cisco) ───────
    enable

        ! ─── Privileged EXEC (mdp : class) ──────
        show ipv6 interface brief
        ! → vérifier statut up/up et adresses correctes

        configure terminal

            ! ─── Global Config ──────────────────
            interface g0/0/1
                ! ─── Interface Config ───────────
                no ipv6 address 2001:db8:c0de:14::1/64
                ! → supprimer l'adresse incorrecte d'abord !
                ipv6 address 2001:db8:c0de:13::1/64
                exit

            end

        show ipv6 route
        ! → vérifier les réseaux directement connectés

        show running-config | include ipv6|interface
        ! → vérifier 2 adresses par interface Gigabit

! ─── Vérification connectivité depuis PC3 ─────────
ping 2001:db8:c0de:13::a

! ─── Vérification connectivité depuis PC4 ─────────
ping 2001:db8:c0de:12::a
```
