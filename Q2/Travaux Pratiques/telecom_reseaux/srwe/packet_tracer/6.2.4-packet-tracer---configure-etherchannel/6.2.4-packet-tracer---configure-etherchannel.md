# Packet Tracer – Configure EtherChannel

> **Objectif** : Configurer des EtherChannels avec PAgP (Cisco) et LACP (IEEE 802.3ad) pour agréger plusieurs liens physiques en un seul lien logique, augmentant la bande passante et la redondance sans créer de boucles STP.

---

## Concepts clés EtherChannel

| Protocole | Standard | Modes compatibles |
|-----------|----------|-------------------|
| **PAgP**  | Cisco propriétaire | `desirable` ↔ `desirable` ou `desirable` ↔ `auto` |
| **LACP**  | IEEE 802.3ad (ouvert) | `active` ↔ `active` ou `active` ↔ `passive` |
| **Static**| Aucun | `on` ↔ `on` |

> ⚠️ `auto` + `auto` (PAgP) = **pas d'EtherChannel**  
> ⚠️ `passive` + `passive` (LACP) = **pas d'EtherChannel**

---

## Port Channel Table

| Channel Group | Ports              | Protocole       |
|---------------|--------------------|-----------------|
| 1             | S1 F0/21–22 / S3 F0/21–22 | PAgP (`desirable`) |
| 2             | S1 G0/1–2 / S2 G0/1–2     | LACP (`active`)    |
| 3             | S2 F0/23–24 / S3 F0/23–24 | LACP négocié (`passive` ↔ `active`) |

## Partie 1 : Configuration de base des switches

### Nommer les switches et vérifier l'état des ports

```
S1# show interfaces | include Ethernet
S1# show interface status
S1# show interfaces trunk
```

### Configurer tous les ports EtherChannel en trunk statique

> ⚠️ Les ports doivent être en **trunk statique** avant de créer les EtherChannels. En mode `dynamic auto` (défaut 2960), ils resteraient en accès et ne rejoindraient pas l'EtherChannel.

**Sur S1 :**

```
S1> enable
S1# configure terminal

S1(config)# hostname S1

S1(config)# interface range f0/21 - 22
S1(config-if-range)# switchport mode trunk
S1(config-if-range)# exit

S1(config)# interface range g0/1 - 2
S1(config-if-range)# switchport mode trunk
S1(config-if-range)# exit

S1(config)# end
```

**Sur S2 :**

```
S2> enable
S2# configure terminal

S2(config)# hostname S2

S2(config)# interface range g0/1 - 2
S2(config-if-range)# switchport mode trunk
S2(config-if-range)# exit

S2(config)# interface range f0/23 - 24
S2(config-if-range)# switchport mode trunk
S2(config-if-range)# exit

S2(config)# end
```

**Sur S3 :**

```
S3> enable
S3# configure terminal

S3(config)# hostname S3

S3(config)# interface range f0/21 - 22
S3(config-if-range)# switchport mode trunk
S3(config-if-range)# exit

S3(config)# interface range f0/23 - 24
S3(config-if-range)# switchport mode trunk
S3(config-if-range)# exit

S3(config)# end
```

Vérifier les trunks actifs :

```
S1# show interfaces trunk
! → F0/21, F0/22, G0/1, G0/2 doivent être en trunking
```

## Partie 2 : EtherChannel PAgP (Port Channel 1 — S1 ↔ S3)

`channel-group 1 mode desirable` active PAgP de façon inconditionnelle — le switch initie la négociation.  
Les deux côtés en `desirable` = EtherChannel PAgP établi ✅

> ⚠️ **Toujours shutdown les ports avant de les ajouter à un channel-group** — sinon EtherChannel Misconfig Guard peut mettre les ports en `err-disabled`.

**Sur S1 :**

```
S1# configure terminal

S1(config)# interface range f0/21 - 22
S1(config-if-range)# shutdown
S1(config-if-range)# channel-group 1 mode desirable
S1(config-if-range)# no shutdown
S1(config-if-range)# exit

S1(config)# interface port-channel 1
S1(config-if)# switchport mode trunk
S1(config-if)# exit

S1(config)# end
```

**Sur S3 :**

```
S3# configure terminal

S3(config)# interface range f0/21 - 22
S3(config-if-range)# shutdown
S3(config-if-range)# channel-group 1 mode desirable
S3(config-if-range)# no shutdown
S3(config-if-range)# exit

S3(config)# interface port-channel 1
S3(config-if)# switchport mode trunk
S3(config-if)# exit

S3(config)# end
```

Vérifier :

```
S1# show etherchannel summary
! → Group 1 : Po1(SU) PAgP  F0/21(P) F0/22(P)
! SU = Layer2, in Use | P = in port-channel
```

## Partie 4 : EtherChannel LACP négocié (Port Channel 3 — S2 ↔ S3)

Ici les deux côtés utilisent des modes **différents** : `passive` (S2) et `active` (S3).

### Vérifier STP sur les ports actifs

STP continue de fonctionner sur les EtherChannels — il traite chaque Port Channel comme un **seul lien logique**.

```
S1# show spanning-tree active
! → Po1, Po2 apparaissent comme interfaces STP
! → Si Po2 est en BLK, il faut forcer S1 comme root bridge
```

Si Po2 (LACP, ports Gigabit) est bloqué par STP, forcer S1 comme root bridge pour rétablir la connectivité :

```
S1# configure terminal

S1(config)# spanning-tree vlan 1 root primary
! → ou équivalent :
S1(config)# spanning-tree vlan 1 priority 24576

S1(config)# end

S1# show spanning-tree active
! → Po1 et Po2 doivent être en FWD ✅
```

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show interfaces trunk` | Vérifier les trunks actifs avant EtherChannel |
| `show interface status` | Vérifier état et mode des ports |
| `channel-group <n> mode <mode>` | Créer le channel group et définir le protocole |
| `interface port-channel <n>` | Accéder à l'interface logique du Port Channel |
| `show etherchannel summary` | Vérifier l'état et le protocole de chaque EtherChannel |
| `show spanning-tree active` | Vérifier l'état STP sur les Port Channels actifs |
| `spanning-tree vlan 1 root primary` | Forcer un switch comme root bridge STP |

## Modes EtherChannel — tableau de compatibilité

| Mode S1        | Mode S2        | Protocole | EtherChannel ? |
|----------------|----------------|-----------|----------------|
| `desirable`    | `desirable`    | PAgP      | ✅ Oui          |
| `desirable`    | `auto`         | PAgP      | ✅ Oui          |
| `auto`         | `auto`         | PAgP      | ❌ Non          |
| `active`       | `active`       | LACP      | ✅ Oui          |
| `active`       | `passive`      | LACP      | ✅ Oui          |
| `passive`      | `passive`      | LACP      | ❌ Non          |
| `on`           | `on`           | Aucun     | ✅ Oui (statique)|
| `on`           | `active/passive`| —        | ❌ Non          |

---

> ⚠️ **Règles importantes EtherChannel** :
> - Tous les ports doivent avoir la **même vitesse et duplex**.
> - Tous les ports doivent être dans le **même VLAN** ou tous configurés en trunk avec le **même VLAN natif**.
> - Toujours **shutdown** les ports **avant** de les ajouter au channel-group.
> - Le numéro du channel-group est **local au switch** — Po1 sur S1 peut se connecter à Po5 sur S2.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Configuration de base ─────────
! ══════════════════════════════════════════════

! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            hostname S1

            interface range f0/21 - 22
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            interface range g0/1 - 2
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show interfaces trunk


! ─── Sur S2 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            hostname S2

            interface range g0/1 - 2
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            interface range f0/23 - 24
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end


! ─── Sur S3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            hostname S3

            interface range f0/21 - 22
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            interface range f0/23 - 24
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end


! ══════════════════════════════════════════════
! ─── PARTIE 2 : PAgP — Port Channel 1 (S1↔S3) ─
! ══════════════════════════════════════════════

! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range f0/21 - 22
                ! ─── Interface Config ────────
                shutdown
                channel-group 1 mode desirable
                no shutdown
                exit

            interface port-channel 1
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show etherchannel summary


! ─── Sur S3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range f0/21 - 22
                ! ─── Interface Config ────────
                shutdown
                channel-group 1 mode desirable
                no shutdown
                exit

            interface port-channel 1
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show etherchannel summary


! ══════════════════════════════════════════════
! ─── PARTIE 3 : LACP — Port Channel 2 (S1↔S2) ─
! ══════════════════════════════════════════════

! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range g0/1 - 2
                ! ─── Interface Config ────────
                shutdown
                channel-group 2 mode active
                no shutdown
                exit

            interface port-channel 2
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show etherchannel summary


! ─── Sur S2 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range g0/1 - 2
                ! ─── Interface Config ────────
                shutdown
                channel-group 2 mode active
                no shutdown
                exit

            interface port-channel 2
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show etherchannel summary


! ══════════════════════════════════════════════
! ─── PARTIE 4 : LACP négocié — Po3 (S2↔S3) ───
! ══════════════════════════════════════════════

! ─── Sur S2 (passive) ─────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range f0/23 - 24
                ! ─── Interface Config ────────
                shutdown
                channel-group 3 mode passive
                no shutdown
                exit

            interface port-channel 3
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end


! ─── Sur S3 (active) ──────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range f0/23 - 24
                ! ─── Interface Config ────────
                shutdown
                channel-group 3 mode active
                no shutdown
                exit

            interface port-channel 3
                ! ─── Interface Config ────────
                switchport mode trunk
                exit

            end

        show etherchannel summary


! ─── STP : forcer S1 comme root si Po2 bloqué ─

! ─── Sur S1 ───────────────────────────────────

    enable

        show spanning-tree active
        ! → Si Po2 en BLK → appliquer :

        configure terminal

            ! ─── Global Config ───────────────
            spanning-tree vlan 1 root primary

            end

        show spanning-tree active
        ! → Po1 et Po2 en FWD ✅
```
