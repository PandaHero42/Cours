# Packet Tracer – Configure Trunks

> **Objectif** : Configurer les liens trunk entre switches pour permettre au trafic de plusieurs VLANs de traverser les liens inter-switches, et configurer un VLAN natif autre que VLAN 1.

---

## Topologie

| Device | Interface | Adresse IP   | Masque        | Port Switch | VLAN |
|--------|-----------|--------------|---------------|-------------|------|
| PC1    | NIC       | 172.17.10.21 | 255.255.255.0 | S2 F0/11    | 10   |
| PC2    | NIC       | 172.17.20.22 | 255.255.255.0 | S2 F0/18    | 20   |
| PC3    | NIC       | 172.17.30.23 | 255.255.255.0 | S2 F0/6     | 30   |
| PC4    | NIC       | 172.17.10.24 | 255.255.255.0 | S3 F0/11    | 10   |
| PC5    | NIC       | 172.17.20.25 | 255.255.255.0 | S3 F0/18    | 20   |
| PC6    | NIC       | 172.17.30.26 | 255.255.255.0 | S3 F0/6     | 30   |

> S1 est le switch central — il connecte S2 via G0/1 et S3 via G0/2. Ces liens doivent être configurés en trunk.

---

## Partie 1 : Vérifier les VLANs

### Étape 1 – Afficher les VLANs sur S1, S2 et S3

Sur **S1** : vérifier que 10 VLANs sont configurés. Tous les 26 ports d'accès sont encore dans VLAN 1 — les trunks n'ont pas encore été configurés.

```
S1> enable
S1# show vlan brief
```

Sur **S2** et **S3** : vérifier que les VLANs sont bien créés et que les ports sont assignés correctement selon le tableau d'adressage.

```
S2# show vlan brief
S3# show vlan brief
```

### Étape 2 – Constater la perte de connectivité inter-switch

PC1 (S2, VLAN 10) et PC4 (S3, VLAN 10) sont dans le même VLAN et le même sous-réseau — mais ne peuvent pas se pinguer.

```
! Depuis PC1
ping 172.17.10.24
! → échoue ✗
```

**Pourquoi ?** Les ports Gigabit qui relient S1–S2 et S1–S3 sont toujours en **VLAN 1 par défaut**. Le trafic VLAN 10 ne peut pas traverser ces liens — les trunks sont nécessaires.

---

## Partie 2 : Configurer les Trunks

### Étape 1 – Configurer le trunking sur S1 avec VLAN 99 comme VLAN natif

`switchport mode trunk` force le port en mode trunk — il transportera le trafic de **tous les VLANs**.  
`switchport trunk native vlan 99` définit le VLAN natif : le trafic non tagué sur ce lien sera associé au VLAN 99 au lieu du VLAN 1 par défaut.  
`interface range g0/1 - 2` permet de configurer les deux interfaces en une seule fois.

```
S1> enable
S1# configure terminal

S1(config)# interface range g0/1 - 2
S1(config-if-range)# switchport mode trunk
S1(config-if-range)# switchport trunk native vlan 99
S1(config-if-range)# exit

S1(config)# end
```

Cliquer sur **Fast Forward Time** pour accélérer la convergence STP.

> ⚠️ **Messages d'erreur attendus** (normaux à cette étape) :
> ```
> %CDP-4-NATIVE_VLAN_MISMATCH: Native VLAN mismatch discovered on
> GigabitEthernet0/2 (99), with S3 GigabitEthernet0/2 (1).
> %CDP-4-NATIVE_VLAN_MISMATCH: Native VLAN mismatch discovered on
> GigabitEthernet0/1 (99), with S2 GigabitEthernet0/1 (1).
> ```
> S1 utilise VLAN 99 comme natif, mais S2 et S3 utilisent encore VLAN 1 → **mismatch**. Ce sera corrigé à l'étape 3.

**Réponse — Pourquoi les pings fonctionnent malgré le mismatch ?**  
Le mismatch concerne uniquement le **VLAN natif** (trafic non tagué). Le trafic des VLANs 10, 20 et 30 est **tagué** avec un en-tête 802.1Q — il traverse correctement le trunk quel que soit le VLAN natif. Les pings entre PCs du même VLAN fonctionnent donc, même si le VLAN natif est incohérent entre les switches.

### Étape 2 – Vérifier le trunking sur S2 et S3

Le protocole **DTP (Dynamic Trunking Protocol)** a automatiquement négocié le mode trunk sur S2 et S3 côté Gigabit, car S1 a été configuré en trunk. On vérifie avec `show interfaces trunk`.

```
S2# show interfaces trunk
S3# show interfaces trunk
```

**Réponse — Quels VLANs actifs sont autorisés sur le trunk ?**  
Par défaut, **tous les VLANs actifs** (1, 10, 20, 30, 99, 150, etc.) sont autorisés à traverser un trunk. La colonne "VLANs allowed and active in management domain" liste les VLANs effectivement présents.

### Étape 3 – Corriger le mismatch de VLAN natif sur S2 et S3

On configure VLAN 99 comme VLAN natif sur les ports Gigabit de S2 et S3 pour correspondre à S1.

**Sur S2 :**

```
S2> enable
S2# configure terminal

S2(config)# interface g0/1
S2(config-if)# switchport trunk native vlan 99
S2(config-if)# exit

S2(config)# end
S2# show interfaces trunk
```

**Sur S3 :**

```
S3> enable
S3# configure terminal

S3(config)# interface g0/1
S3(config-if)# switchport trunk native vlan 99
S3(config-if)# exit

S3(config)# end
S3# show interfaces trunk
```

Les messages de mismatch doivent disparaître.

### Étape 4 – Vérifier la configuration finale

Vérifier que le VLAN natif est bien 99 sur les interfaces trunk :

```
S2# show interfaces g0/1 switchport
! → chercher "Trunking Native Mode VLAN: 99"

S2# show vlan
```

**Réponse — Pourquoi G0/1 sur S2 n'est-il plus assigné à VLAN 1 ?**  
Parce que G0/1 est maintenant en **mode trunk**. Un port trunk n'appartient à aucun VLAN d'accès — il transporte le trafic tagué de tous les VLANs. Il ne figure donc plus dans la liste des ports du VLAN 1 dans `show vlan`. Le VLAN natif (99) gère le trafic non tagué, mais le port n'est pas "membre" d'un VLAN spécifique comme un port d'accès.

---

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show vlan brief` | Affiche VLANs, statut et ports associés |
| `show interfaces trunk` | Affiche les ports en mode trunk et les VLANs autorisés |
| `show interfaces <int> switchport` | Affiche les détails de configuration d'un port |
| `interface range g0/1 - 2` | Configure plusieurs interfaces simultanément |
| `switchport mode trunk` | Force le port en mode trunk (transporte tous les VLANs) |
| `switchport trunk native vlan <num>` | Définit le VLAN natif du trunk (trafic non tagué) |

---

> ⚠️ **À retenir** :
> - Un **port trunk** transporte le trafic de **tous les VLANs** simultanément grâce au tagging 802.1Q.
> - Le **VLAN natif** est le seul VLAN dont le trafic n'est **pas tagué** sur le trunk. Il doit être le **même des deux côtés** du lien pour éviter les problèmes de sécurité et les erreurs CDP.
> - **DTP** peut négocier automatiquement le trunk si un côté est configuré en trunk, mais il est recommandé de le configurer explicitement des deux côtés.

---

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Vérification ──────────────────
! ══════════════════════════════════════════════

    ! ─── Sur S1 ──────────────────────────────
    enable
        show vlan brief
        ! → 10 VLANs, tous les ports dans VLAN 1

    ! ─── Sur S2 et S3 ────────────────────────
    enable
        show vlan brief
        ! → VLANs configurés, ports assignés ok

! ─── Depuis PC1 : ping vers PC4 ───────────────
ping 172.17.10.24
! → échoue : liens inter-switch pas encore en trunk


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Configuration des trunks ──────
! ══════════════════════════════════════════════

! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface range g0/1 - 2
                ! ─── Interface Config ────────
                switchport mode trunk
                switchport trunk native vlan 99
                exit

            end

        ! → Fast Forward Time pour accélérer STP
        ! → Messages NATIVE_VLAN_MISMATCH attendus (normal)


! ─── Sur S2 ───────────────────────────────────

    enable

        show interfaces trunk
        ! → vérifier DTP a négocié le trunk automatiquement

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport trunk native vlan 99
                exit

            end

        show interfaces trunk
        show interfaces g0/1 switchport
        show vlan


! ─── Sur S3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                switchport trunk native vlan 99
                exit

            end

        show interfaces trunk
        show interfaces g0/1 switchport
        show vlan


! ─── Vérification finale depuis PC1 ───────────
ping 172.17.10.24
! → réussit ✅ (trunk actif, VLAN 10 traverse S1)
```
