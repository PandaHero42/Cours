# Packet Tracer – HSRP Configuration Guide

> **Objectif** : Configurer HSRP (Hot Standby Router Protocol) pour fournir une passerelle par défaut redondante aux hôtes du LAN — si le routeur actif tombe, le routeur standby prend automatiquement le relai sans intervention.

---

## Concepts clés HSRP

| Terme | Définition |
|-------|-----------|
| **FHRP** | First Hop Redundancy Protocol — famille de protocoles de redondance de passerelle |
| **HSRP** | Hot Standby Router Protocol — implémentation Cisco du FHRP |
| **Routeur actif** | Traite tout le trafic des hôtes vers la gateway virtuelle |
| **Routeur standby** | En attente — prend le relai si le routeur actif devient indisponible |
| **IP virtuelle** | Adresse partagée configurée comme passerelle sur les hôtes |
| **Priorité** | Valeur 0–255, défaut = 100. La plus haute = routeur actif |
| **Preempt** | Permet au routeur actif de reprendre son rôle quand il revient en ligne |
| **Groupe HSRP** | Numéro identifiant le groupe — doit être identique sur les deux routeurs |

## Partie 1 : Vérifier la connectivité initiale

### Tracer le chemin vers le Web Server

Depuis **PC-A** :
```
tracert 209.165.200.226
```
**Réponse — chemin depuis PC-A :**  
PC-A → R1 (192.168.1.1) → R2 → I-Net → Web Server

Depuis **PC-B** :
```
tracert 209.165.200.226
```
**Réponse — chemin depuis PC-B :**  
PC-B → R3 (192.168.1.3) → R2 → I-Net → Web Server

### Observer le comportement sans HSRP quand R3 tombe

1. Supprimer le câble entre R3 et S3 (**outil Delete** dans la barre).
2. Depuis PC-B : `tracert 209.165.200.226`

**Réponse — résultat :**  
La trace échoue — PC-B ne peut plus joindre le Web Server. Sans HSRP, la perte du routeur actif = perte totale de connectivité pour les hôtes de ce LAN.

3. Recâbler S3 G0/2 → R3 G0/0 (Copper Straight-Through).

## Partie 2 : Configurer HSRP

### Étape 1 – Configurer R1 comme routeur actif

`standby version 2` active HSRPv2 qui supporte IPv4 et IPv6 (v1 = IPv4 uniquement).  
`standby 1 ip` définit l'IP virtuelle partagée — c'est cette adresse que les hôtes utiliseront comme passerelle.  
`standby 1 priority 150` rend R1 actif (150 > 100 par défaut de R3).  
`standby 1 preempt` permet à R1 de reprendre son rôle actif automatiquement après un retour en ligne.

```
R1> enable
R1# configure terminal

R1(config)# interface g0/1
R1(config-if)# standby version 2
R1(config-if)# standby 1 ip 192.168.1.254
R1(config-if)# standby 1 priority 150
R1(config-if)# standby 1 preempt
R1(config-if)# exit

R1(config)# end
```

**Réponse — quelle sera la priorité HSRP de R3 ?**  
**100** — c'est la valeur par défaut. R3 n'ayant pas de priorité configurée manuellement, il sera automatiquement standby.

### Étape 2 – Configurer R3 comme routeur standby

R3 utilise uniquement `standby version 2` et `standby 1 ip` — pas de priorité ni de preempt car il doit rester standby.

```
R3> enable
R3# configure terminal

R3(config)# interface g0/0
R3(config-if)# standby version 2
R3(config-if)# standby 1 ip 192.168.1.254
R3(config-if)# exit

R3(config)# end
```

### Étape 3 – Vérifier la configuration HSRP

```
R1# show standby
! → State is Active
! → Virtual IP address is 192.168.1.254
! → Priority 150
! → Preemption enabled
! → Standby router is 192.168.1.3

R3# show standby
! → State is Standby
! → Virtual IP address is 192.168.1.254
! → Priority 100 (default)
! → Preemption disabled
! → Active router is 192.168.1.1
```

Version courte :
```
R1# show standby brief
R3# show standby brief
```

**Réponses aux questions :**

- **Quel routeur est actif ?** → R1 (priorité 150 > 100)
- **Quelle est l'adresse MAC virtuelle ?** → `0000.0C9F.F001` (adresse MAC virtuelle HSRPv2 groupe 1)
- **IP et priorité du standby ?** → R3 : 192.168.1.3, priorité 100

### Étape 4 – Mettre à jour la passerelle des hôtes

Changer la passerelle par défaut de **PC-A, PC-B, S1 et S3** vers l'IP virtuelle HSRP.

**Réponse — quelle adresse utiliser ?**  
`192.168.1.254` — l'IP virtuelle HSRP. C'est cette adresse qui reste stable même si R1 ou R3 tombe.

Via GUI pour les PCs — **Desktop → IP Configuration** :
- PC-A : Gateway → `192.168.1.254`
- PC-B : Gateway → `192.168.1.254`

Via CLI pour les switches :
```
S1# configure terminal
S1(config)# ip default-gateway 192.168.1.254
S1(config)# end

S3# configure terminal
S3(config)# ip default-gateway 192.168.1.254
S3(config)# end
```

## Partie 3 : Observer le fonctionnement HSRP

### Étape 1 – Tracer le chemin après configuration HSRP

Depuis **PC-B** :
```
tracert 209.165.200.226
```

**Réponse — le chemin diffère-t-il d'avant HSRP ?**  
Non — le trafic passe toujours par R1 (actif) car la gateway virtuelle `192.168.1.254` est gérée par R1. Le chemin est le même, mais maintenant la passerelle est l'IP virtuelle au lieu de l'IP physique.

### Étape 2 – Simuler la panne de R1

1. **Supprimer le câble** entre R1 et S1 (outil Delete).
2. Depuis PC-B : `tracert 209.165.200.226` — répéter jusqu'à voir le chemin complet.

**Réponse — quelle différence ?**  
Après un délai de convergence HSRP (~10 secondes), R3 prend le rôle actif. Le trafic passe maintenant par R3 au lieu de R1 — mais les hôtes continuent d'utiliser `192.168.1.254` comme passerelle sans aucune modification.

### Étape 3 – Rétablir le lien R1–S1

1. Recâbler S1 → R1 G0/1 (Copper Straight-Through).
2. Depuis PC-B : `tracert 209.165.200.226`

**Réponse — quel chemin est utilisé après retour de R1 ?**  
Grâce à `preempt`, R1 reprend automatiquement son rôle actif — le chemin revient via R1.

**Réponse — sans `preempt`, que se serait-il passé ?**  
R3 serait resté actif même après le retour de R1. R1 resterait standby car sans `preempt`, un routeur qui revient en ligne ne reprend pas automatiquement son rôle actif même si sa priorité est plus haute.

---

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `standby version 2` | Active HSRPv2 (supporte IPv4 et IPv6) |
| `standby <grp> ip <ip>` | Définit l'IP virtuelle du groupe HSRP |
| `standby <grp> priority <val>` | Définit la priorité (défaut 100, plus haute = actif) |
| `standby <grp> preempt` | Reprend le rôle actif automatiquement au retour |
| `show standby` | Affiche le détail HSRP (rôle, IP virtuelle, priorité) |
| `show standby brief` | Résumé HSRP en une ligne par groupe |

---

> ⚠️ **À retenir** :
> - La **même IP virtuelle** doit être configurée sur R1 et R3 et sur tous les hôtes comme passerelle.
> - Le **numéro de groupe** doit être identique sur les deux routeurs.
> - Sans `preempt`, le routeur actif ne revient pas automatiquement après une panne.
> - HSRPv2 support IPv4 et IPv6 — v1 supporte uniquement IPv4.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Vérification initiale ─────────
! ══════════════════════════════════════════════

! PC-A > tracert 209.165.200.226  → chemin via R1
! PC-B > tracert 209.165.200.226  → chemin via R3

! Supprimer câble R3–S3 (outil Delete)
! PC-B > tracert 209.165.200.226  → échoue (pas de HSRP)
! Recâbler S3 G0/2 ↔ R3 G0/0


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Configuration HSRP ────────────
! ══════════════════════════════════════════════

! ─── Sur R1 (routeur actif) ───────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/1
                ! ─── Interface Config ────────
                standby version 2
                standby 1 ip 192.168.1.254
                standby 1 priority 150
                standby 1 preempt
                exit

            end

        show standby
        show standby brief


! ─── Sur R3 (routeur standby) ─────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface g0/0
                ! ─── Interface Config ────────
                standby version 2
                standby 1 ip 192.168.1.254
                ! → pas de priority ni preempt → reste standby
                exit

            end

        show standby
        show standby brief


! ─── Mise à jour des gateways ─────────────────

! ─── Sur S1 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            ip default-gateway 192.168.1.254

            end


! ─── Sur S3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            ip default-gateway 192.168.1.254

            end


! ─── PC-A et PC-B : Desktop > IP Configuration ─
! Gateway : 192.168.1.254 (IP virtuelle HSRP)


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Test HSRP ─────────────────────
! ══════════════════════════════════════════════

! PC-B > tracert 209.165.200.226  → via R1 (actif)

! Supprimer câble R1–S1 (outil Delete)
! PC-B > tracert 209.165.200.226  → via R3 (nouveau actif après ~10s)

! Recâbler S1 ↔ R1 G0/1
! PC-B > tracert 209.165.200.226  → via R1 (reprend grâce à preempt)
```
