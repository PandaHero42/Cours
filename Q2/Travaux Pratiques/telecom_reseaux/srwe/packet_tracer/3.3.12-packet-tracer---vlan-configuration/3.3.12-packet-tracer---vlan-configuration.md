# Packet Tracer – VLAN Configuration

> **Objectif** : Créer et nommer des VLANs sur des switches, assigner des ports d'accès aux VLANs, et comprendre pourquoi les trunks sont nécessaires pour la communication inter-switch entre VLANs.

---

## Partie 1 : Vérifier la configuration VLAN par défaut

### Étape 1 – Afficher les VLANs actuels sur S1

Par défaut, **toutes les interfaces** sont assignées au VLAN 1 et tous les ports sont actifs dans ce VLAN.  
La commande `show vlan brief` affiche la liste des VLANs, leur statut et les ports associés.

```
S1> enable
S1# show vlan brief
```

### Étape 2 – Vérifier la connectivité initiale

Avant toute configuration VLAN, les PCs du même sous-réseau peuvent se pinguer (ils sont tous dans VLAN 1) :

```
! Depuis PC1 (Desktop > Command Prompt)
ping 172.17.10.24
! → réussit (PC4, même réseau /24)

! Depuis PC2
ping 172.17.20.25
! → réussit (PC5, même réseau /24)

! Depuis PC3
ping 172.17.30.26
! → réussit (PC6, même réseau /24)
```

**Réponse — Quels avantages les VLANs apportent-ils ?**  
Les VLANs permettent de segmenter logiquement le réseau sans matériel supplémentaire, d'améliorer la sécurité en isolant les groupes d'utilisateurs, de réduire le trafic broadcast, et de simplifier l'administration en regroupant les utilisateurs par fonction plutôt que par emplacement physique.

## Partie 2 : Créer et nommer les VLANs

### Étape 1 – Créer les VLANs sur S1

> ⚠️ Les noms sont **sensibles à la casse** et doivent correspondre exactement aux valeurs demandées — une erreur de casse = points perdus.

La commande `vlan <numéro>` crée le VLAN et entre en mode `config-vlan`.  
La commande `name` lui donne un nom lisible.

```
S1> enable
S1# configure terminal

S1(config)# vlan 10
S1(config-vlan)# name Faculty/Staff
S1(config-vlan)# exit

S1(config)# vlan 20
S1(config-vlan)# name Students
S1(config-vlan)# exit

S1(config)# vlan 30
S1(config-vlan)# name Guest(Default)
S1(config-vlan)# exit

S1(config)# vlan 99
S1(config-vlan)# name Management&Native
S1(config-vlan)# exit

S1(config)# vlan 150
S1(config-vlan)# name VOICE
S1(config-vlan)# exit

S1(config)# end
```

### Étape 2 – Vérifier la configuration VLAN

`show vlan brief` affiche uniquement le nom, le statut et les ports — c'est la commande demandée.

```
S1# show vlan brief
```

## Partie 3 : Assigner les VLANs aux ports

### Étape 1 – Assigner les ports sur S2

`switchport mode access` configure le port en mode accès (pour un seul VLAN, connecté à un PC).  
`switchport access vlan <num>` assigne ce port au VLAN souhaité.

```
S2> enable
S2# configure terminal

S2(config)# interface f0/11
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 10
S2(config-if)# exit

S2(config)# interface f0/18
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 20
S2(config-if)# exit

S2(config)# interface f0/6
S2(config-if)# switchport mode access
S2(config-if)# switchport access vlan 30
S2(config-if)# exit

S2(config)# end
S2# show vlan brief
```

### Étape 2 – Assigner les ports sur S3

Mêmes assignations de ports que S2.

```

S3(config)# interface f0/11

S3(config-if)# switchport access vlan 10
S3(config-if)# exit

S3(config)# interface f0/18
S3(config-if)# switchport mode access

S3(config-if)# switchport mode access
```

### Étape 3 – Configurer le VLAN voix sur S3 F0/11

Le port F0/11 de S3 connecte **à la fois** un PC (PC4, VLAN 10) et un **téléphone IP Cisco**.  
Le téléphone IP contient un mini-switch 3 ports intégré : un port vers le switch S3, un port vers PC4, et un port interne pour les fonctions voix.

Deux choses à configurer :
- `switchport voice vlan 150` : le trafic voix du téléphone sera tagué VLAN 150.
- `mls qos trust cos` : le switch fait confiance aux valeurs QoS (CoS) définies par le téléphone IP pour prioriser le trafic voix — indispensable pour la qualité d'appel.

```
S3(config)# interface f0/11
S3(config-if)# mls qos trust cos
S3(config-if)# switchport voice vlan 150
S3(config-if)# exit

S3(config)# end
S3# show vlan brief
```

### Étape 4 – Constater la perte de connectivité

Après assignation des ports aux VLANs, tenter de pinguer PC4 depuis PC1 :

```
! Depuis PC1
ping 172.17.10.24
! → échoue ✗
```

**Réponse — Pourquoi les pings échouent-ils maintenant ?**  
Les ports d'accès sont bien assignés aux bons VLANs, mais les **liens entre switches** (Gig0/1 sur S2) sont toujours assignés à VLAN 1 par défaut. Le trafic VLAN 10 de S2 ne peut pas traverser vers S3 car le lien inter-switch n'est pas configuré en **trunk** — il ne transporte qu'un seul VLAN (VLAN 1). Les trames taggées VLAN 10 sont donc bloquées.

**Réponse — Comment résoudre ce problème ?**  
Configurer les ports inter-switches en **mode trunk** (`switchport mode trunk`) pour qu'ils transportent le trafic de tous les VLANs simultanément. C'est l'objet du prochain Packet Tracer.

---

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show vlan brief` | Affiche nom, statut et ports de tous les VLANs |
| `vlan <num>` | Crée un VLAN et entre en mode config-vlan |
| `name <nom>` | Nomme le VLAN (sensible à la casse) |
| `switchport mode access` | Configure le port en mode accès (1 seul VLAN) |
| `switchport access vlan <num>` | Assigne le port à un VLAN |
| `switchport voice vlan <num>` | Assigne un VLAN voix au port (téléphonie IP) |
| `mls qos trust cos` | Active la confiance QoS pour le trafic voix |

---

> ⚠️ **À retenir** : Créer un VLAN et nommer ses ports ne suffit pas pour la communication inter-switch. Les liens entre switches doivent être configurés en **trunk** pour transporter plusieurs VLANs. Sans trunk, chaque lien ne transporte que VLAN 1.

## Script complet (toutes les commandes dans l'ordre)

```
! ══════════════════════════════════════════════
! ─── PARTIE 1 : Vérification initiale ─────────
! ══════════════════════════════════════════════

    ! ─── Sur S1 ──────────────────────────────
    enable

        ! ─── Privileged EXEC ─────────────────
        show vlan brief
        ! → tous les ports dans VLAN 1 par défaut

! ─── Vérification connectivité (depuis PCs) ───
! PC1 > ping 172.17.10.24   → réussit (PC4)
! PC2 > ping 172.17.20.25   → réussit (PC5)
! PC3 > ping 172.17.30.26   → réussit (PC6)


! ══════════════════════════════════════════════
! ─── PARTIE 2 : Création des VLANs ────────────
! (Répéter sur S1, S2 et S3)
! ══════════════════════════════════════════════

    ! ─── User EXEC ───────────────────────────
    enable

        ! ─── Privileged EXEC ─────────────────
        configure terminal

            ! ─── Global Config ───────────────
            vlan 10
                ! ─── VLAN Config ─────────────
                name Faculty/Staff
                exit

            vlan 20
                ! ─── VLAN Config ─────────────
                name Students
                exit

            vlan 30
                ! ─── VLAN Config ─────────────
                name Guest(Default)
                exit

            vlan 99
                ! ─── VLAN Config ─────────────
                name Management&Native
                exit

            vlan 150
                ! ─── VLAN Config ─────────────
                name VOICE
                exit

            end

        show vlan brief
        ! → vérifier les 5 VLANs créés (sans ports encore)


! ══════════════════════════════════════════════
! ─── PARTIE 3 : Assignation des ports ─────────
! ══════════════════════════════════════════════

! ─── Sur S2 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface f0/11
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                exit

            interface f0/18
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 20
                exit

            interface f0/6
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            end

        show vlan brief


! ─── Sur S3 ───────────────────────────────────

    enable

        configure terminal

            ! ─── Global Config ───────────────
            interface f0/11
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 10
                mls qos trust cos
                switchport voice vlan 150
                ! → données PC4 = VLAN 10, voix IP Phone = VLAN 150
                exit

            interface f0/18
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 20
                exit

            interface f0/6
                ! ─── Interface Config ────────
                switchport mode access
                switchport access vlan 30
                exit

            end

        show vlan brief


! ─── Vérification finale ──────────────────────
! PC1 > ping 172.17.10.24
! → échoue : liens inter-switches pas encore en trunk !
```
