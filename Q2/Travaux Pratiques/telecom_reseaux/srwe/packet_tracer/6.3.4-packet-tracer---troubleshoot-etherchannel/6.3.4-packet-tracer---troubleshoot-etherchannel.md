# Packet Tracer – Troubleshoot EtherChannel

> **Objectif** : Identifier et corriger les erreurs de configuration EtherChannel sur 4 switches — problèmes de mode trunk, mauvaise assignation de ports aux channel groups, et protocoles incorrects.

---

## Topologie — Port Channel Table

| Channel Group | Ports                        | Protocole attendu |
|---------------|------------------------------|-------------------|
| 1             | S1 G0/1–G0/2 / S3 G0/1–G0/2 | LACP active       |
| 2             | S2 G0/1–G0/2 / S4 G0/1–G0/2 | LACP active       |
| 3             | S1 F0/23–F0/24 / S2 F0/23–F0/24 | LACP active   |
| 4             | S3 F0/23–F0/24 / S4 F0/23–F0/24 | LACP active   |
| 5             | S1 F0/21–F0/22 / S4 F0/21–F0/22 | LACP active   |
| 6             | S2 F0/21–F0/22 / S3 F0/21–F0/22 | LACP active   |

> ⚠️ **Exigence** : Tous les EtherChannels doivent utiliser **LACP mode active** des deux côtés.

---

## Méthodologie de dépannage EtherChannel

```
1. Vérifier le mode trunk des ports physiques   → show interfaces trunk
2. Vérifier les assignations aux channel groups  → show etherchannel summary
3. Vérifier le protocole utilisé                 → show etherchannel summary
4. Corriger dans l'ordre : trunk → assignation → protocole
```

## Partie 1 : Vérifier et corriger les modes trunk

### Diagnostic

Sur chaque switch, vérifier que tous les ports EtherChannel sont bien en trunk :

```
S1# show interfaces trunk
S1# show interface status
! → chercher des ports en mode "access" au lieu de "trunk"
```

### Problème type — port en mode access

Si un port est en `access` au lieu de `trunk`, l'EtherChannel ne peut pas se former correctement — tous les ports d'un même channel group doivent avoir la même configuration trunk.

### Correction — Sur chaque switch concerné

```
S1(config-if-range)# exit
```


Comparer le résultat avec le **Device Table** — vérifier que chaque port est dans le bon channel group.

### Problème type — port dans le mauvais channel group

Si un port est assigné au mauvais channel group (ex: F0/21 dans le groupe 3 au lieu du groupe 5), il faut d'abord le **retirer** du groupe incorrect, puis le **réassigner** au bon groupe.

### Comment retirer un port d'un channel group

Pour retirer un port d'un channel group, utiliser `no channel-group` :

```
S1(config)# interface range f0/21 - 22
S1(config-if-range)# shutdown
S1(config-if-range)# no channel-group
S1(config-if-range)# channel-group 5 mode active
S1(config-if-range)# no shutdown
S1(config-if-range)# exit
```

## Partie 3 : Vérifier et corriger le protocole

### Diagnostic

```
S1# show etherchannel summary
! → colonne Protocol : vérifier que tous les groupes sont en LACP (pas PAgP ni "on")
! → les ports doivent être en mode "active" (A) et non "passive" (P) ou "on"
```

### Problème type — protocole incorrect

Si un port est en PAgP (`desirable`/`auto`) ou en mode statique (`on`) au lieu de LACP `active`, il faut reconfigurer le channel-group.

### Correction type — remplacer PAgP par LACP

```
S1(config)# interface range f0/21 - 22
S1(config-if-range)# shutdown
S1(config-if-range)# no channel-group 5
S1(config-if-range)# channel-group 5 mode active
S1(config-if-range)# no shutdown
S1(config-if-range)# exit
```

### Vérification finale

```
S1# show etherchannel summary
S2# show etherchannel summary
S3# show etherchannel summary
S4# show etherchannel summary
```

Résultat attendu pour chaque switch — tous les groupes doivent afficher :
- **SU** = Layer2, in Use
- **P** sur chaque port = in port-channel
- **LACP** comme protocole

```
! Exemple de sortie correcte sur S1 :
! Group  Port-channel  Protocol    Ports
! ------+-------------+-----------+----------------
! 1      Po1(SU)       LACP        G0/1(P) G0/2(P)
! 3      Po3(SU)       LACP        F0/23(P) F0/24(P)
! 5      Po5(SU)       LACP        F0/21(P) F0/22(P)
```

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show etherchannel summary` | Vue globale des EtherChannels : protocole, ports, état |
| `show interfaces trunk` | Vérifier le mode trunk des ports physiques |
| `show interface status` | Vérifier vitesse, duplex, mode de chaque port |
| `no channel-group` | Retirer un port de son channel group |
| `channel-group <n> mode active` | Assigner un port à un channel group en LACP actif |

---

## Problèmes classiques EtherChannel — tableau de référence

| Problème | Symptôme dans `show etherchannel summary` | Correction |
|----------|-------------------------------------------|------------|
| Port en mode access | Port marqué `I` (stand-alone) | `switchport mode trunk` |
| Mauvais channel group | Port dans le mauvais groupe | `no channel-group` puis réassigner |
| Protocole incorrect | `PAgP` au lieu de `LACP` | Reconfigurer `channel-group mode active` |
| Mode passive des deux côtés | EtherChannel ne monte pas | Mettre au moins un côté en `active` |
| Ports de vitesses différentes | Port marqué `u` (unsuitable) | Uniformiser la vitesse |

---

> ⚠️ **Rappel** : Toujours **shutdown** les ports avant de modifier leur channel-group. Un port `err-disabled` peut être réactivé avec `shutdown` puis `no shutdown`.
