# Packet Tracer – Investigate STP Loop Prevention

> **Objectif** : Observer le fonctionnement du Spanning Tree Protocol (STP) dans un réseau redondant — identifier le root bridge, les états des ports, et observer la convergence STP après une panne de lien.

---

## Concepts clés STP

| Terme | Définition |
|-------|-----------|
| **Root Bridge** | Switch élu comme racine de l'arbre — tous les chemins sont calculés par rapport à lui |
| **Root Port (RP)** | Port d'un switch non-root qui offre le meilleur chemin vers le root bridge → état **FWD** |
| **Designated Port (DP)** | Port sur chaque segment qui transmet les trames vers/depuis le root → état **FWD** |
| **Blocked Port (BLK)** | Port mis en blocage pour éviter les boucles → n'envoie ni reçoit de données |
| **FWD (Forwarding)** | Port actif — transmet les trames normalement → voyant **vert** |
| **BLK (Blocking)** | Port bloqué par STP — ne transmet rien → voyant **orange** |

> 💡 **Principe** : STP élit un Root Bridge, puis bloque les ports redondants pour créer une topologie **sans boucle** tout en maintenant une **redondance physique**. Si un lien actif tombe, STP reconverge et débloque un port précédemment bloqué.

---

## Partie 1 : Observer une instance STP convergée

### Étape 1 – Vérifier la connectivité

```
! Depuis PC1 (Desktop > Command Prompt)
ping <IP de PC2>
! → réussit ✅
```

### Étape 2 – Afficher l'état STP sur chaque switch

Sur **S1, S2 et S3**, exécuter la commande :

```
S1# show spanning-tree vlan 1
S2# show spanning-tree vlan 1
S3# show spanning-tree vlan 1
```

> 💡 Ne regarder que les ports **Gigabit (G0/1 et G0/2)** — les ports FastEthernet sont des ports d'accès vers les PCs et ne font pas partie du spanning tree inter-switch.

### Tableau d'état STP (à compléter dans Packet Tracer)

| Switch | Port | Statut attendu | Root Bridge ? |
|--------|------|----------------|---------------|
| S1     | G0/1 | FWD            | À vérifier    |
| S1     | G0/2 | FWD            | À vérifier    |
| S2     | G0/1 | FWD            | À vérifier    |
| S2     | G0/2 | FWD ou BLK     | À vérifier    |
| S3     | G0/1 | FWD            | À vérifier    |
| S3     | G0/2 | **BLK**        | Non           |

> Le switch avec le **Bridge ID le plus bas** (priorité + MAC) est élu Root Bridge.  
> Tous ses ports sont des **Designated Ports** → état FWD.  
> Sur les autres switches, **un port redondant sera bloqué** pour casser la boucle.

### Réponses aux questions

**Que signifie le voyant orange sur un lien ?**  
Le voyant **orange** indique qu'un port est dans un état STP autre que Forwarding — généralement **Blocking**. Le port est physiquement connecté mais STP l'a mis en blocage pour éviter une boucle. Aucune trame de données n'y transite.

**Quel chemin prennent les trames de PC1 à PC2 ?**  
Les trames empruntent le chemin qui ne passe **pas par le port bloqué**. Si G0/2 de S3 est bloqué, le chemin passe par S1 → S2 (ou S2 → S1) sans transiter par S3 entre les deux switches principaux.

**Pourquoi les trames ne passent-elles pas par S3 ?**  
STP a bloqué un port de S3 pour éliminer la boucle formée par les trois switches. S3 est toujours connecté physiquement, mais le port bloqué empêche la création d'un chemin redondant actif qui causerait une tempête de broadcast.

**Pourquoi STP a-t-il mis un port en état Blocking ?**  
Sans STP, les trames broadcast circuleraient indéfiniment dans la boucle formée par S1–S2–S3 (**broadcast storm**), saturant le réseau. STP bloque un port pour garantir qu'il n'existe qu'**un seul chemin actif** entre deux points du réseau, tout en gardant le lien physique disponible en cas de panne.

## Partie 2 : Observer la convergence STP

### Étape 1 – Ouvrir le CLI de S3 et supprimer le lien S1–S2

1. Ouvrir une fenêtre CLI sur **S3** et laisser tourner :
   ```
   S3# show spanning-tree vlan 1
   ```

2. Dans Packet Tracer, sélectionner l'outil **Delete** (croix rouge) dans la barre d'outils.
3. Cliquer sur le **câble qui relie S1 et S2** pour le supprimer.

### Étape 2 – Observer la convergence en temps réel

Revenir rapidement sur le CLI de S3 et répéter la commande :

```
S3# show spanning-tree vlan 1
```

Appuyer sur la **flèche ↑** pour rappeler la commande et la lancer en boucle jusqu'à ce que le voyant orange sur le câble S3 devienne **vert**.

**Observer l'évolution du statut de G0/2 sur S3 :**

```
! Séquence de convergence STP observée :
! 1. BLK  (Blocking)   → port bloqué, lien S1-S2 encore actif
! 2. LIS  (Listening)  → S1-S2 supprimé, STP détecte le changement
! 3. LRN  (Learning)   → port apprend les adresses MAC
! 4. FWD  (Forwarding) → port actif, nouveau chemin établi ✅
```

> 💡 **La convergence STP prend environ 30–50 secondes** en mode classique (STP 802.1D) à cause des timers : **Listening** dure 15s, **Learning** dure 15s. **Rapid STP (RSTP 802.1w)** converge en quelques secondes.

**Réponse — Que se passe-t-il sur G0/2 de S3 pendant la convergence ?**  
Le port passe progressivement de **BLK → LIS → LRN → FWD**. STP détecte la perte du lien S1–S2, recalcule la topologie, et débloque le port de S3 précédemment bloqué pour rétablir la connectivité via un chemin alternatif.

### Étape 3 – Vérifier la connectivité après convergence

```
! Depuis PC1
ping <IP de PC2>
! → réussit ✅ (chemin recalculé via S3)
```

**Réponse — Y a-t-il encore des ports en orange ?**  
Non. Après suppression du lien S1–S2, il n'y a plus de chemin redondant dans la topologie — tous les liens restants sont nécessaires pour maintenir la connectivité. STP n'a donc plus besoin de bloquer aucun port : tous les ports Gigabit passent en **FWD** (vert).

## Résumé des commandes clés

| Commande | Rôle |
|----------|------|
| `show spanning-tree vlan 1` | Affiche l'état STP : root bridge, ports, statuts |
| `show spanning-tree vlan 1 detail` | Version détaillée avec timers et coûts |

---

## Résumé des états STP

```
Physiquement connecté mais STP actif :
  BLK (Blocking)   → reçoit les BPDUs, n'envoie pas de données  → voyant orange
  LIS (Listening)  → écoute les BPDUs, prépare la convergence   → voyant orange
  LRN (Learning)   → apprend les MACs, ne transmet pas encore   → voyant orange
  FWD (Forwarding) → transmet normalement                       → voyant vert
  DIS (Disabled)   → port administrativement désactivé          → pas de voyant
```

---

> ⚠️ **Conclusion** : STP garantit une topologie **sans boucle** en bloquant les ports redondants. Si un lien actif tombe, STP **reconverge automatiquement** en débloquant un port précédemment bloqué — assurant la redondance sans boucle. Le délai de convergence (30–50s pour 802.1D) peut être réduit avec **Rapid STP (RSTP)**.
