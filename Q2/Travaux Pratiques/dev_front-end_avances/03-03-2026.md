# git (à détailler après pour les différents points)

- Logiciel de gestion de versions, permet de stocker un ensemble de fichiers en conservant une cronologie au niveau des modifs.

### Avantages

- 1. **Traçabilité (Versioning)**
  - **Historisation et persistance**
  - **Retour en arrière (Rollback)**
  - **Analyse différentielle**

- Git utilise une base de données non relationnelle de type key/value.

- 2. **Collaboration et parallélisation du travail**
  - Git permet de passer d'un boulot individuel à une collaboration complexe.
  - **Dév non-linéaire (Branching)**
  - **Isolation des flux**
  - **Synchronisation distribuée**
- 3. **Coordination et écosystème de dépoloiment**
  - Le dépôt distant comme pivot du workflow professionnel.
  - **Gouvernance du code (Code Review)**
  - **Source unique de vérité**
  - **Automatisation du déploiment (CI/CD)**

### Système de Stockage 

- On utilise du SHA1, juste besoin d'éviter les collisions. Clé-valeur.
