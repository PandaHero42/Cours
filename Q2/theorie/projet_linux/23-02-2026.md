# Les partages Samba

### Introduction : 

**Samba** est une implémentation libre du protocole **SMB/CIFS (Server Message Block / Common Internet File System)** qui permet le **partage de fichiers et d'imprimantes** entre machines Linux/Unix et Windows sur un réseau local.

Concrètement, Samba permet à un serveur Linux d'apparaître comme un serveur de fichiers Windows. Un poste Windows peut donc accéder à un dossier partagé Linux comme s'il s'agissait d'un partage Windows classique (\\\serveur\partage).

Samba repose principalement sur le protocole **SMB**, utilisé nativement par les systèmes Windows sur :

- le partage de fichiers
- le partage d'imprimantes
- l'authentification réseau
- l'intégration à un domaine (Active Directory)

## Prérequis :

- Une machine Linux (Debian, Ubuntu, CentOS, etc.)
- Accès administrateur (``sudo`` ou ``root``)
- Réseau fonctionnel (IP fixe pour le serveur)
- Les utilisateurs Samba (pour les permissions)
- Les permissions sur les dossiers (pour accéder aux permissions)
- Un firewall qui ne bloque pas !

*Remarque :* Ports utilisés par le firewall ?

| Port  | Protocole | Usage |
|------|-----|------------|
| 137 | UDP  | NetBIOS  |
| 138   | UDP | NetBIOS   |
| 139  | TCP  | SMB ancien |
| 139  | TCP | SMB direct |

- Le port 445 c'est le port vraiment important aujourd'hui.

## Installation :

Installation typique sous les distribution pouvant utiliser des binary compatible avec REHL.

```bash
# --------------- MISE A JOUR DU SYSTEME ------------------------

sudo dnf update
sudo dnf install samba samba-common samba-client

# ------ ACTIVATION ET DEMARRAGE DES SERVICES AU DEMARRAGE ------

sudo systemctl enable smb.service
sudo systemctl enable nmb.service
sudo systemctl start smb
sudo systemctl start nmb

# ----------------- CONFIGURATION DU FIREWALL -------------------

sudo firewall-cmd --permanent --add-service=samba
sudo firewall-cmd --reload
sudo firewall-cmd --list-services # Vérification

# --------------- SAUVEGARDE DE LA CONFIGURATION ----------------

cd /etc/samba
sudo cp smb.conf smb.conf.backup # Toujours sauvegarder avant modification
```

## Configuration : 

### Globalement dans le serveur :

```bash
# -------------- CREATION DU DOSSIER DE PARTAGE ---------------

sudo mkdir -p /srv/partages/petitPartage # -p crée toute l'arborescence

# ---------- CREATION DU GROUPE ET DES UTILISATEURS -----------

sudo groupadd testGroup # groupe dédié au partage

sudo useradd -m test
sudo useradd -m test2

sudo usermod -aG testGroup test # ajout au groupe secondaire
sudo usermod -aG testGroup test2

groups test # vérification
groups test2

# ------------- AJOUT DES UTILISATEURS A SAMBA ----------------

sudo smbpasswd -a test # mot de passe Samba
sudo smbpasswd -a test2

sudo smbpasswd -e test # activation du compte
sudo smbpasswd -e test2

# ------------ CONFIGURATION DES PERMISSIONS LINUX ------------

sudo chown root:testGroup /srv/partages/petitPartage
sudo chmod 2775 /srv/partages/petitPartage

# 2 = SGID -> héritage automatique du groupe sinon caca niveau permissions
# 775 = rwxrwxr-x
```

## Le fichier de configuration : 

#### Normalement il n'est pas sensible aux espaces mais certains fichiers le sont (genre avant et après l'égal)...

```bash
[petitPartage]
    comment = Un partage
    path = /srv/partages/petitPartage
    browseable = yes
    writable = yes
    valid users = @testGroup
    force group = testGroup
    create = 0664
    directory mask = 2775
```

ça c'était la base

## Aspect "sécurité" :

```bash
# ça se mets dans le [global] dans le fichier /etc/samba/smb.conf
# --------------------------- LOGS -----------------------------

log file = /var/log/samba/log.%m # Un fichier de log par machine cliente (%m = nom NetBIOS)
max log size = 1000 # Taille max d'un log en Ko (ici 1 Mo)
log level = 2 # Niveau de verbosité (1 = minimal, 10 = debut très détaillé)

# ---------------------- CONTROLE D'ACCES ---------------------

map to guest = never # Empêche toute conversion automatique vers l'utilisateur guest
# => Si authentification échoue -> accès refusé

hosts allow = 192.168.100.0/24 # Autorise uniquement les hôtes de ce réseau
hosts deny = 0.0.0.0/0 # Refuse tout le reste (principe du "deny all")

# ---------------------- SECURITE PROTOCOLE -------------------

server min protocol = SMB3 # Force le serveur à utiliser au minimum SMB3
client min protocol = SMB3 # Force le client Samba local à utiliser SMB3 minimum
```

## Redémarrage, vérification et tests :

```bash
# ------------------ VERIFICATION DE LA CONFIG ----------------

sudo testparm

# ------------------ REDEMARRAGE DES SERVICES -----------------

sudo systemctl restart smb
sudo systemctl restart nmb

# ------------------ TESTS CÔTÉ CLIENT ------------------------

# Liste des partages

smbclient -L //<IP_SERVEUR>/ -U test

# Connexion au partage
smbclient //<IP_SERVEUR>/petitPartage -U test

# Et là, on arrive dans un shell et on peut essayer :

ls
put fichier.txt
get fichier.txt

# et c'est good
```

# Les serveurs Web

## Introduction :

Un **serveur web** est un logiciel qui **répond aux requêtes des clients** (souvent des navigateurs web) et leur fournit des **pages web, fichiers ou applications** via le protocole HTTP ou HTTPS.
Il écoute sur un ou plusieurs port(80 pour HTTP, 443 pour HTTPS) et gère les demandes simultanées des utilisateurs ou les deux ports et les faire passer de l'un à l'autre.

En pratique, le serveur web fait deux choses principales :

1. **Recevoir les requêtes** des clients.
2. **Envoyer une réponse**, souvent sous forme de pages HTML, images, fichiers ou données dynamiques générées par des scripts (PHP, Python, etc.).

## Les deux grands noms : 
1. **Apache HTTP Server**
   - Le serveur web le plus connu et utilisé historiquement.
   - Très modulable grâce à ses **modules** : gestion SSL, PHP, réécriture d'URL, d'authentification, ...
   - et même SMTP
2. **Nginx**
   - Plus récent et très performant pour gérer **beaucoup de connexions simultanées.**
   - Fonctionne aussi souvent comme **reverse proxy** et permet pleins de choses par défaut.

## Installation :

```bash
# --------------------- INSTALLATION D'APACHE ----------------------

sudo dnf install httpd

# --------------------- ACTIVATION + DEMARRAGE ---------------------

sudo systemctl enable --now httpd

# --nom = active le service au boot ET le démarre immédiatement

# ------------------- CONFIGURATION DU FIREWALL --------------------

sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

# ---------------------- VERIFICATION ------------------------------

sudo firewall-cmd --list-services # vérifie que http/https sont bien ouverts
sudo systemctl status httpd # vérifie que le service tourne

# ---------------------- TESTS -------------------------------------

curl -I http://<IP_SERVEUR> # Test en ligne de commande (header HTTP)
# ou accès via navigateur depuis le réseau interne
```
